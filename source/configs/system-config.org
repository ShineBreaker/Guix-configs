--------------

此系统配置的关键在于 *让系统能够正常启动* ，所以会尽量精简软件包以及相关服务的数量

待系统成功安装并进入系统之后，则可以直接手动应用用户配置

--------------

* Agent 指引

系统配置修改指引。

本区域为 AI 助手提供 Home 配置的上下文和决策规则。阅读此区域后再进行修改。

** 系统配置职责：
+ 定义 operating-system 的组成部分，包括引导器、文件系统、内核、用户、系统包、系统服务、skeleton 文件。
+ 这些组件共同确保系统能够正常启动和运行。

** 关键组件说明：
+ 引导器使用 Limine（由 rosenthal 频道提供），替代传统的 GRUB。
+ 文件系统采用 LUKS 加密 + Btrfs 子卷 + tmpfs 混合架构，实现根目录重启清空但数据持久化的效果。
+ 内核使用 主线内核（由 nonguix 频道提供），用于现代设备的正常启动。

** 文件系统架构：
+ 根目录使用 tmpfs（重启后清空），Btrfs 子卷用于持久化数据，/data 分区通过 bind-mount 挂载到用户目录。
+ 具体的子卷映射和持久化目录定义在 information.scm 中，包括 %btrfs-subvolumes 和 %data-dirs 变量。

** source/files/ 用途：
+ 存放静态配置模板文件，如 nftables.conf 防火墙规则。
+ 这些文件通过 local-file 引用到系统配置中，在系统构建时被复制到目标位置。

** 修改约定：
+ 系统层变更优先修改本文件中的对应代码块。生成的 tmp/system-config.scm 是自动导出的结果，不应手动编辑。
+ 全局变量如用户名、Btrfs 配置等统一在 information.scm 中定义，本文件通过 (load "../source/information.scm") 加载。

** 决策规则：
+ 本文件仅放置系统级必需内容，包括引导配置、内核设置、文件系统挂载、系统服务定义。
+ 用户级工具、桌面环境配置、个人偏好设置应放入 Home 配置（home-config.org）。

(define %mapped-devices-config
  (list (mapped-device
         (source (uuid "327f2e02-1e4f-48b2-87f0-797c481850c9"))
         (target "root")
         (type luks-device-mapping))))

(define %tmpfs
  (list (file-system
         (mount-point "/")
         (device "tmpfs")
         (type "tmpfs")
         (flags '(no-atime))
         (options "mode=0755,nr_inodes=1m,size=10%")
         (check? #f))
        (file-system
         (mount-point "/tmp")
         (device "tmpfs")
         (type "tmpfs")
         (options "mode=1777,nr_inodes=1m,size=75%")
         (create-mount-point? #t)
         (check? #f))
        (file-system
         (device "tmpfs")
         (mount-point "/var/lock")
         (type "tmpfs")
         (flags '(no-suid no-dev strict-atime))
         (options "mode=1777,nr_inodes=800k,size=20%")
         (create-mount-point? #t)
         (check? #f))))

(define %persist-filesystem
  (map (match-lambda
         ((subvol mount-point)
          (file-system
           (device (file-system-label "Linux"))
           (mount-point mount-point)
           (type "btrfs")
           (dependencies %mapped-devices-config)
           (options (string-append "subvol=" subvol ",compress=zstd:6"))
           (create-mount-point? #t)
           (needed-for-boot? #t)
           (check? (string=? mount-point "/gnu"))))) %btrfs-subvolumes))

(define %data-filesystem
  (file-system
   (device (file-system-label "Linux"))
   (mount-point "/data")
   (type "btrfs")
   (dependencies %mapped-devices-config)
   (options (string-append "subvol=" %btrfs-subvol-data ",compress=zstd:6"))
   (create-mount-point? #t)))

(define %bind-mounts
  (map (lambda (dirs)
         (file-system
          (mount-point (string-append "/home/" username "/" dirs))
          (device (string-append "/data/" dirs))
          (type "none")
          (flags '(bind-mount))
          (dependencies (list %data-filesystem))
          (create-mount-point? #t))) %data-dirs))

(define %file-systems-config
  (append %bind-mounts %persist-filesystem %tmpfs
          (list %data-filesystem
                (file-system
                 (device (uuid "9699-52A2"
                               'fat))
                 (mount-point "/efi")
                 (type "vfat")
                 (create-mount-point? #t))) %base-file-systems))

** 文件系统相关服务
在这里就是一些负责维护文件系统树的服务

+ *fixed-machine-id* 负责固定 =/etc/machine-id=
  这个文件中的内容，具体的算法在 [[../information.scm][information.scm]]
  中
+ *fix-data-perms* 负责修复 =/data= 目录下所有文件夹的权限
+ *create-xdg-dirs* 负责在 =/data= 目录和 =/home/user=
  目录中创建同名的文件夹，以用于bind-mount

#+NAME: filesystem-services
#+begin_src scheme
(list (simple-service 'fixed-machine-id etc-service-type
                      (list `("machine-id" ,(plain-file "machine-id"
                                                        fixed-machine-id))))

      (simple-service 'fix-data-perms activation-service-type
                      #~(begin
                          (use-modules (guix build utils))
                          (mkdir-p "/data")
                          (chmod "/data" #o1777)))

      (simple-service 'create-xdg-dirs activation-service-type
                      (with-imported-modules (source-module-closure '((guix
                                                                       build
                                                                       utils)))
                                             #~(begin
                                                 (use-modules (guix build
                                                                    utils))
                                                 (let* ((pw (getpwnam #$username))
                                                        (uid (passwd:uid pw))
                                                        (gid (passwd:gid pw))
                                                        (home (string-append
                                                               "/home/"
                                                               #$username)))
                                                   (for-each (lambda (dir)
                                                               (let ((path (string-append
                                                                            home
                                                                            "/"
                                                                            dir)))
                                                                 (mkdir-p
                                                                  path)
                                                                 (chown path
                                                                        uid gid)
                                                                 (chmod path
                                                                        #o755)))
                                                             '#$%data-dirs))))))

负责处理 *用户骨架 (/etc/skel) 目录下的文件

该目录下的所有文件都会在用户第一次创建时写入到用户的目录中

适合在此放置一些需要经常修改的模板类服务

#+NAME: skeletons-config
#+begin_src scheme
(map (lambda (path)
       `(,path ,(local-file (string-append "source/files/skel/" path))))
     '(".config/mihomo/config.yaml" ".config/git-credential-keepassxc"))
#+end_src

* Users
用户信息

=password= 行可以用于设置默认密码

利用 =(password (crypt "InitialPassword!" "$6$abc")))=
可以创建一个初始密码，其中crypt函数的第一个字符串为 *明文的密码*

然后在进入系统之后重新注释，并自行设置密码，或者将 *password*
后的函数设置为 *Hash化后的密码*

利用
#+begin_src bash
echo "InitialPassword!" | guix shell openssl -+ openssl passwd -6 -stdin
#+end_src

即可返回需要填入的值
