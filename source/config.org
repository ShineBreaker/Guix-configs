* Agent 指引
** System
职责： *operating-system* 层。引导器（Limine）、文件系统（LUKS+Btrfs+tmpfs）、内核（nonguix linux-7.0）、八个按职责拆分的服务块、用户账户。

全局变量定义在 =source/information.scm= （username、%btrfs-subvolumes、%data-dirs、guix-channels）。

=source/files/= 存放需路径注入的静态模板（nftables.conf、rounded.qss、zed.json、skel/），走 home-files-service-type。

原则：系统层最小化，能用 Home 解决不上浮到 System。

** Home

职责： *home-environment* 层。4 个包列表（desktop/terminal/devtools/user）、dotfiles 部署（home-dotfiles-service-type，layout stow 从 dotfiles/enable/<app>/ 软链接到 ~）、Home 服务、环境变量、字体。

=source/files/= （home-files-service-type + 路径注入 $$bin/foo$$）与 =dotfiles/enable/= （stow 软链接）为两套独立部署路径，不互通。

新增应用流程：
1. =dotfiles/enable/<app>/= 建目录
2. 加到包列表
3. 加到 =dotfile-services packages= 列表。

Emacs 新包必须同时在 =emacs-services= 的 ~specifications->manifest~ 中声明。
=~/.config/<app>/= 指向 */gnu/store 只读副本* ，改源后必须 ~blue home~ 重建软链接才生效。

** 核心概念
+ *Org Noweb* ：#+NAME 命名块 → <<ref>> 引用 → tangle 为 tmp/config.scm。<<ref>> 是 Org Mode 语法，非 Scheme
+ =config.org= 是唯一 Org 源， *不要新建第二个* ； *不要手动编辑 tmp/ 下的产物*

** 工作方法
*** 块级编辑（优先于读全文件）
修改 config.org 中单个 #+NAME: 块时，用 =block-*= 任务避免读 2000 行：

1. ~blue block-show <name>~ → 提取块 body 到 tmp/block-<name>.scm（并打印路径）
2. ~tail -n +3 <文件>~ → 编辑 body（前两行是 lang= 和 noweb/plain 标记）
3. ~blue block-replace <name> <body-file>~ → 原子写回 + 括号验证

*注意* ：body 可能含 <<ref>> 占位，需自行追踪引用块。括号验证只对 scheme 块触发。
高耦合大块（emacs-services 等）仍建议 read 整段。失败后手动 git checkout source/config.org 恢复。

*** 验证流程
+ ~blue check~ → 逐块括号平衡检查（最快，能定位到具体出错的块）
+ ~blue --dry-run rebuild~ → 完整构建但不写入系统（tangle/括号检查真跑，reconfigure 短路）
+ ~blue home~ → 仅 Home 层生效，不需 sudo（首选调试方式）
+ Agent *禁止* 自行 ~blue rebuild~ / ~guix system reconfigure~ （需 sudo，卡住 CLI）

*** 决策规则
+ 优先改 =dotfiles/enable/<app>/= ，只当需要 Guix 介入时才改 config.org
+ 新增/重命名 dotfile 子目录后必须同步更新 =dotfile-services packages= 列表
+ 能用 Home 解决就不用 System
+ 修改后必走验证流程

(define %persist-filesystem
  (map (match-lambda
         ((subvol mount-point)
          (file-system
           (device (file-system-label "Linux"))
           (mount-point mount-point)
           (type "btrfs")
           (dependencies %mapped-devices-config)
           (options (string-append "subvol=" subvol ",compress=zstd:6"))
           (create-mount-point? #t)
           (needed-for-boot? #t)
           (check? (string=? mount-point "/gnu"))))) %btrfs-subvolumes))

(define %data-filesystem
  (file-system
   (device (file-system-label "Linux"))
   (mount-point "/data")
   (type "btrfs")
   (dependencies %mapped-devices-config)
   (options (string-append "subvol=" %btrfs-subvol-data ",compress=zstd:6"))
   (create-mount-point? #t)))

(define %bind-mounts
  (map (lambda (dirs)
         (file-system
          (mount-point (string-append "/home/" username "/" dirs))
          (device (string-append "/data/" dirs))
          (type "none")
          (flags '(bind-mount))
          (dependencies (list %data-filesystem))
          (create-mount-point? #t))) %data-dirs))

(define %file-systems-config
  (append %bind-mounts %persist-filesystem %tmpfs
          (list %data-filesystem
                (file-system
                 (device (uuid "9699-52A2"
                               'fat))
                 (mount-point "/efi")
                 (type "vfat")
                 (create-mount-point? #t))) %base-file-systems))

(define %swap-devices
  (list (swap-space
         (target (uuid "169557cc-00a4-448c-9bb6-7bd80fc2b023")))))

** Services
*** 文件系统相关服务
在这里就是一些负责维护文件系统树的服务

+ *fixed-machine-id* 负责固定 =/etc/machine-id= 这个文件中的内容，具体的算法在 [[../information.scm][information.scm]] 中
+ *fix-data-perms* 负责修复 =/data= 目录下所有文件夹的权限
+ *create-xdg-dirs* 负责在 =/data= 目录和 =/home/user= 目录中创建同名的文件夹，以用于bind-mount

#+NAME: filesystem-services
#+begin_src scheme
(list (simple-service 'fixed-machine-id etc-service-type
                      (list `("machine-id" ,(plain-file "machine-id"
                                                        fixed-machine-id))))

      ;; 每次启动时将 swap 分区写入 /sys/power/resume（休眠必需）
      (simple-service 'setup-resume-device
                      shepherd-root-service-type
                      (list (shepherd-service
                             (provision '(resume-device))
                             (requirement '(file-systems))
                             (one-shot? #t)
                             (start #~(lambda ()
                                        (call-with-output-file "/sys/power/resume"
                                          (lambda (port)
                                            (display "259:3" port))))))))

      (simple-service 'fix-data-perms activation-service-type
                      #~(begin
                          (use-modules (guix build utils))
                          (mkdir-p "/data")
                          (chmod "/data" #o1777)))

      (simple-service 'create-xdg-dirs activation-service-type
                      (with-imported-modules (source-module-closure '((guix
                                                                       build
                                                                       utils)))
                                             #~(begin
                                                 (use-modules (guix build
                                                                    utils))
                                                 (let* ((pw (getpwnam #$username))
                                                        (uid (passwd:uid pw))
                                                        (gid (passwd:gid pw))
                                                        (home (string-append
                                                               "/home/"
                                                               #$username)))
                                                   (for-each (lambda (dir)
                                                               (let ((path (string-append
                                                                            home
                                                                            "/"
                                                                            dir)))
                                                                 (mkdir-p
                                                                  path)
                                                                 (chown path
                                                                        uid gid)
                                                                 (chmod path
                                                                        #o755)))
                                                             '#$%data-dirs))))))
#+end_src

*** 软件自启动

这里利用 *scheme* 的 =lambda= 函数来包装了一个列表;

这样的操作在我的配置文件里面多次出现，这样可以不用重复写很大一段内容。

这也正是 Guix 的优势所在: 可以利用 Scheme 强大的生态来进行各种操作。

#+NAME: home-shepherd-services
#+begin_src scheme
(let* ((make-shepherd-service
        (lambda* (pkg path requirement)
                 (shepherd-service (provision (list (string->symbol (package-name pkg))))
                                   (requirement requirement)
                                   (start #~(lambda args
                                              ((make-forkexec-constructor
                                                (list #$(file-append pkg path))
                                                #:environment-variables (environ)
                                                #:log-file (string-append (getenv "HOME") "/.local/state/shepherd/"
                                                                          #$(package-name pkg) ".log")))
                                              args))
                                   (stop #~(make-kill-destructor))
                                   (respawn? #t)))))
  (list (simple-service 'essential-desktop-services home-shepherd-service-type
                        (list (make-shepherd-service kanata "/bin/kanata" '(graphical-session))
                              (make-shepherd-service kdeconnect "/bin/kdeconnect-indicator" '(graphical-session))
                              (make-shepherd-service poweralertd "/bin/poweralertd" '(graphical-session))
                                        ; (make-shepherd-service swayidle "/bin/swayidle" '(graphical-session))
                              (make-shepherd-service xdg-desktop-portal "/libexec/xdg-desktop-portal" '(graphical-session))
                              (make-shepherd-service xdg-desktop-portal-gtk "/libexec/xdg-desktop-portal-gtk" '(graphical-session))
                              (make-shepherd-service xdg-desktop-portal-wlr "/libexec/xdg-desktop-portal-wlr" '(graphical-session))))

        ;; Hermes Agent —— 把 dashboard backend 和 messaging gateway 做成 shepherd 用户级守护进程，
        ;; 与 NixOS nixosModules.hermes-agent 的 systemd 服务语义一致，但走 Shepherd 管理。
        ;; 这样 hermes-desktop / hermes chat / messaging platforms 共享同一个常驻 backend，
        ;; 不再依赖 desktop session 在线。
        (simple-service 'hermes-services home-shepherd-service-type
                        (list (shepherd-service
                               (provision '(hermes-backend))
                               (requirement '(graphical-session))
                               (start #~(lambda args
                                          ((make-forkexec-constructor
                                            (list (string-append (getenv "HOME") "/.nix-profile/bin/hermes")
                                                  "dashboard" "--no-open"
                                                  "--host" "127.0.0.1" "--port" "9119")
                                            #:environment-variables (environ)
                                            #:log-file (string-append (getenv "HOME")
                                                                      "/.local/state/shepherd/hermes-backend.log"))
                                           args)))
                               (stop #~(make-kill-destructor))
                               (respawn? #t))

                              (shepherd-service
                               (provision '(hermes-gateway))
                               (requirement '(hermes-backend))
                               (start #~(lambda args
                                          ((make-forkexec-constructor
                                            (list (string-append (getenv "HOME") "/.nix-profile/bin/hermes")
                                                  "gateway" "run")
                                            #:environment-variables (environ)
                                            #:log-file (string-append (getenv "HOME")
                                                                      "/.local/state/shepherd/hermes-gateway.log"))
                                           args)))
                               (stop #~(make-kill-destructor))
                               (respawn? #t))))))
