* 核心概念速览
在深入配置前，先理解几个 Guix 特有的概念：

** 1. Channel（频道）
Guix 的软件包定义分布在多个「频道」中。本配置除默认 =guix= 外还包含：

+ [[https://gitlab.com/nonguix/nonguix][nonguix]] — Linux 内核、专有驱动等
+ [[https://github.com/ShineBreaker/jeans][jeans]] — 个人频道（自定义包）
+ [[https://codeberg.org/hako/rosenthal][rosenthal]] — Window Manager 增强组件（limine 引导器、`computed-substitution-with-inputs` 等）

频道定义在 [[./source/channel.scm][source/channel.scm]]，版本锁定在 [[./source/channel.lock][source/channel.lock]]。 =channel.lock= 利用 channel 参数中 =commit= 字段锁住具体 commit，从而固定仓库。

** 2. System + Home（统一配置）
单一 =guix system reconfigure= 同时激活系统配置与用户配置（用户配置由 =guix-home-service= 内嵌提供）。Org 源文件 =source/config.org= 中包含系统段与用户段两套 Noweb 代码块。

*原则*：能用 Home 解决就不用 System，保持系统层最小化。

** 3. Org Mode 撰写配置
配置写在 =.org= 文件中，代码嵌入在 =src= 代码块里：

+ 配置和文档在一起
+ 用 Emacs 的 =org-babel-tangle= 提取代码
+ 阅读体验更好

本仓库使用 =blue=（基于 blue-mono 的 Scheme DSL）自动处理 Org 文件的导出。

* 仓库架构详解

#+begin_example
.
├── blueprint.scm           # 任务运行器（基于 blue-mono Scheme DSL）
├── source/                 # 配置源码
│   ├── AGENTS.md           # 本目录 AI 助手指引
│   ├── channel.scm         # 频道定义
│   ├── channel.lock        # 频道版本锁定
│   ├── information.scm     # 全局变量（用户名、Btrfs、%data-dirs 等）
│   ├── config.org          # ★ 唯一的 Org 配置源（system + home 全部 Noweb 拼合）
│   ├── files/              # 静态模板（nftables.conf、rounded.qss、zed.json、skel/）
│   └── nix/                # Nix home-manager 备份分支（与 Guix 不互通）
├── dotfiles/
│   ├── enable/             # 启用的配置（统一由 Guix Home 的 home-dotfiles-service-type 部署）
│   │   ├── agents/         # Agent 资产（Pi、Crush、KB、loopctl、共享 skills）
│   │   ├── desktop/        # 桌面环境（niri、autostart、xdg-portal、xfce4 helpers）
│   │   ├── emacs/          # Emacs 配置（独立 git 子模块）
│   │   ├── system/         # 系统级（containers、pipewire、xdg user-dirs）
│   │   ├── terminal/       # 终端工具链（fish、tmux、foot、btop、starship、broot、fastfetch）
│   │   └── utilities/      # 开发工具（helix、git、kanata、pnpm、winapps、rime、gnupg）
│   └── disable/            # 已弃用的旧配置（保留参考）
├── tools/                  # 辅助工具
├── screenshots/            # 预览截图
└── tmp/                    # blue 生成的中间产物（自动生成，勿手动编辑）
#+end_example

** 关键入口

| 文件 / 目录                              | 用途                                              | 修改频率                   |
|------------------------------------------+---------------------------------------------------+----------------------------|
| =source/information.scm=                 | 用户名、Btrfs 子卷、持久化目录 =%data-dirs=       | 初次使用时必改             |
| =source/config.org=                      | 系统 + 用户配置（唯一 Org 源）                    | 经常                       |
| =dotfiles/enable/*=                      | 启用的 dotfiles 配置                              | 经常                       |
| =blueprint.scm=                          | blue 任务定义                                     | 需要添加自定义任务时修改   |
| =source/channel.scm=                     | 频道定义                                          | 添加/删除频道时            |

* 构建管线

#+begin_example
source/config.org
        │
        ▼  blue rebuild → Emacs org-babel-tangle（Noweb 拼合）
tmp/config.scm
        │
        ▼  guix time-machine --channels=source/channel.lock system reconfigure
单一 reconfigure：operating-system 同时声明 guix-home-service（含 home-environment）

* 快速开始
** 1. 安装系统（新机器）
#+begin_src sh
# 进入 Guix 安装环境后，克隆本仓库
git clone https://codeberg.org/BrokenShine/Guix-configs

# 修改 source/information.scm 中的用户名等基本信息
# 然后初始化系统
guix shell blue -- blue init
#+end_src

** 2. 日常更新（已安装系统）
#+begin_src sh
# 查看所有可用命令
blue --list

# 常用命令
blue rebuild            # tangle + 括号检查 + system reconfigure + locate --update
blue update             # 更新 channel.lock 并签名提交
blue pull               # guix pull --allow-downgrades --fallback

# 验证配置（不实际应用）
GUIX_DRY_RUN=1 blue rebuild

# 括号平衡检查（最轻量，不触发 guix）
blue check
#+end_src

* 可复用组件指南
以下组件设计为可独立复用，你可以直接复制到自己的配置中。

** ✅ 完全独立可复用
*** 1. 内核优化参数（[[./source/config.org::*Kernel][source/config.org]] 内 =kernel-arguments= 代码块）

#+begin_src scheme
(kernel-arguments
 '("zswap.enabled=1"
   "zswap.compressor=zstd"
   "zswap.max_pool_percent=20"
   "zswap.zpool=zsmalloc"
   "net.ipv4.tcp_congestion_control=bbr"
   "net.ipv4.tcp_fastopen=3"
   "vm.vfs_cache_pressure=50"
   "kernel.sched_autogroup_enabled=1"))
#+end_src

*如何复用*：直接复制到 =operating-system= 的 =kernel-arguments= 字段。

*** 2. 静态配置文件（[[./source/files/][source/files/]]）

+ =nftables.conf= — nftables 防火墙规则
+ =rounded.qss= — Qt 圆角样式
+ =zed.json= — Zed 编辑器配置
+ =skel/= — 骨架文件模板（mihomo config）

*如何复用*：直接复制文件内容，或在自己的配置中使用 =(local-file "...")= 引用。

*** 3. Org 模式配置工作流

本仓库的核心创新：用 Org 文件管理配置，通过 =blue= 自动提取。

#+begin_src scheme
;; blueprint.scm 中的关键函数
(define (rebuild)
  "tangle → 括号检查 → guix time-machine → system reconfigure"
  ...)
#+end_src

*如何复用*：

1. 复制 =blueprint.scm=，并安装 =blue=（blue-mono）
2. 将 =.scm= 配置改写为 =.org= 格式
3. 用 =#+begin_src scheme :tangle ...= 包裹代码块
4. 用 Noweb =<<ref>>= 引用子块

** ⚠️ 需要适配后复用
*** 1. Btrfs + tmpfs 混合架构（[[./source/information.scm][source/information.scm]]）
实现「重启后根目录清空，但数据持久化」的不可变系统效果（具体需要持久化的目录与 Btrfs 结构都需要对应修改）：

#+begin_src scheme
;; 定义需要持久化的数据目录
(define %data-dirs
  '(".local/share/gnupg"
    ".local/share/keyrings"
    ".local/share/fish"
    "Documents"
    "Downloads"
    ...))

;; 定义 Btrfs 子卷映射
(define %btrfs-subvolumes
  '(("SYSTEM/Guix/@boot"   "/boot")
    ("SYSTEM/Guix/@data"   "/var/lib")
    ("SYSTEM/Guix/@gnu"    "/gnu")
    ("DATA/Home/Guix"      "/home")
    ...))
#+end_src

*如何复用*：参考 =information.scm= 中的 =%data-dirs= 和 =%btrfs-subvolumes= 定义，配合 =config.org= 中的 Btrfs 挂载配置。

*** 2. Dotfiles 组织方式（[[./dotfiles/][dotfiles/]]）

采用 *统一 Stow 部署*：所有 =enable/<dir>/= 子目录由 Guix Home 的 =home-dotfiles-service-type=（layout ='stow=）部署为软链接，不再有 immutable / mutable 拆分。

#+begin_example
dotfiles/enable/
├── agents/.config/pi/                 → ~/.config/pi/
├── agents/.config/agents/             → ~/.config/agents/
├── agents/.config/crush/              → ~/.config/crush/
├── agents/.config/loopctl/            → ~/.config/loopctl/
├── agents/.local/bin/                 → ~/.local/bin/
├── desktop/.config/niri/              → ~/.config/niri/
├── desktop/.config/autostart/         → ~/.config/autostart/
├── system/.config/containers/         → ~/.config/containers/
├── system/.config/pipewire/           → ~/.config/pipewire/
├── terminal/.config/fish/             → ~/.config/fish/
├── terminal/.config/tmux/             → ~/.config/tmux/
├── utilities/.config/helix/           → ~/.config/helix/
├── utilities/.config/git/             → ~/.config/git/
└── emacs/.config/emacs/               → ~/.config/emacs/
#+end_example

*如何复用*：在 =config.org= 中声明：

#+begin_src scheme
(service home-dotfiles-service-type
         (home-dotfiles-configuration
          (directories '("../dotfiles/enable"))
          (layout 'stow)
          (packages '("agents" "desktop" "emacs"
                      "noctalia-suite" "system" "terminal" "utilities"))))
#+end_src

新增 / 修改 =enable/<dir>/= 下文件后运行 =blue rebuild= 重新链接。

*** 3. tmux 配置（[[./dotfiles/enable/terminal/.config/tmux/][dotfiles/enable/terminal/.config/tmux/]]）
tmux 配置采用 Guile 驱动 + Bash 胶水的混合架构，核心设计：

+ *侧边栏导航树*：浮动 pane 渲染的层级树（Session → Window → Pane），前缀键 =Ctrl+x B= 切换
+ *无插件架构*：所有功能通过外部脚本 + tmux hooks 实现，无 TPM 依赖
+ *信号驱动刷新*：操作触发 SIGUSR1 即时重绘，30 秒心跳兜底
+ *CJK 感知渲染*：侧栏文本使用 Guile 实现的中文宽度计算
+ *工作区快照*：支持将当前 session 的窗口布局保存/恢复为 JSON（tmuxifier）

目录结构：

#+begin_example
~/.config/tmux/
├── tmux.conf                       # 主配置
└── scripts/
    ├── sidebar-toggle              # 侧边栏生命周期（Bash）
    ├── sidebar-render.scm          # 数据采集 + 渲染（Guile）
    ├── tmux-helpers.scm            # 共享工具模块（缓存、锁、CJK 宽度）
    ├── session-selector            # 会话选择器（fzf）
    ├── window-jump                 # 窗口跳转（fzf）
    └── which-key                   # 快捷键帮助弹窗
#+end_example

*如何复用*：复制 =tmux.conf= 与 =scripts/= 到自己的配置。需要 tmux 3.3a+ 和 Guile；Guix 用户可在 home 环境中添加 =guile= 包。

*注意*：侧栏依赖 foot 终端行为，在其他终端下可能需要调整 =default-terminal= 与 =terminal-overrides=。

* 配置分层策略
理解何时修改哪个文件：

** 添加新软件包
1. *仅用户需要*（游戏、开发工具、媒体播放器）→ 在 =source/config.org= 的 home-packages 段（=desktop-packages-list= / =terminal-packages-list= / =devtools-packages-list= / =user-packages-list=）添加
2. *系统级必需*（文件系统工具、固件、limine 等）→ 在 =source/config.org= 的 system-packages 段（=packages-list=）添加

** 添加新服务
1. *用户服务*（如 swayidle、mako）→ =source/config.org= 的 home-services 段
2. *系统服务*（如 nftables、docker）→ =source/config.org= 的 system-services 段

** 添加配置文件
*优先放入 =dotfiles/enable/<app>/=*：

1. 创建子目录（如 =dotfiles/enable/<app>/.config/<app>/=）
2. 在 =source/config.org= 的 =dotfile-services= 块的 =packages= 列表中追加目录名
3. =blue rebuild= 重新链接

#+begin_src sh
# 示例：添加应用配置
mkdir -p dotfiles/enable/app-name/.config/app-name
cp ~/app-config.toml dotfiles/enable/app-name/.config/app-name/
# 编辑 source/config.org 的 dotfile-services.packages，加上 "app-name"
blue rebuild
#+end_src

*仅在需要动态生成时使用 =source/files/=*（如需注入 Guix 软件包路径）。

* 常用任务
** 修改用户名
编辑 =source/information.scm=：

#+begin_src scheme
(define username "your-username")
#+end_src

然后运行 =blue rebuild= 。

** 添加 Btrfs 持久化目录
编辑 =source/information.scm=：

#+begin_src scheme
(define %data-dirs
  '(... "your-new-dir"))
#+end_src

如需新增 Btrfs 子卷挂载，同步编辑 =%btrfs-subvolumes=。然后在 =config.org= 的 Btrfs 配置部分添加对应的 bind-mount。

** 添加新的 Channel
编辑 =source/channel.scm= 添加频道定义，然后运行：

#+begin_src sh
blue update
#+end_src

** 自定义 blue 任务
编辑 =blueprint.scm= 添加新函数：

#+begin_src scheme
(define (my-task)
  "我的自定义任务"
  (log-info "执行自定义任务")
  ($ `("echo" "Hello, Guix!")))
#+end_src

然后运行 =blue my-task= 。

* 故障排除
** 配置声明与目录不一致
确保 =dotfiles/enable/<name>/= 目录存在，且与 =source/config.org= 中 =home-dotfiles-configuration= 声明的 =packages= 一致。

** 系统启动后数据丢失
检查 =source/information.scm= 中的 =%data-dirs= 与 Btrfs 子卷配置是否正确；确保 =/data= 分区已正确挂载。

** Org 导出失败
确保安装了 Emacs 且启用了 =org-babel-tangle=：

#+begin_src sh
emacs --batch -l org --eval "(require 'ob-tangle)"
#+end_src

** 配置语法检查
修改 ~*.org~ 配置后，可用 =dry-run= 模式验证语法（不会实际应用到系统）：

#+begin_src sh
GUIX_DRY_RUN=1 blue rebuild
#+end_src

=blue rebuild= 在执行前会自动运行括号平衡检查。也可手动单独检查：

#+begin_src sh
blue check
#+end_src
