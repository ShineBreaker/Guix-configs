** 核心概念速览
在深入配置前，先理解几个 Guix 特有的概念：

*** 1. Channel（频道）
Guix 的软件包定义分布在多个「频道」中。默认只有 =guix= 官方频道，本配置额外添加了：

+ *[[https://gitlab.com/nonguix/nonguix][nonguix]]* - 提供 Linux内核、专有驱动等
+ *[[https://github.com/ShineBreaker/jeans][jeans]]* - 个人频道，包含自定义包
+ *[[https://codeberg.org/gofranz/panther.git][pantherx]]* - 提供一些需要用到的工具
+ *[[https://codeberg.org/hako/rosenthal][rosenthal]]* - 提供很多用于强化 window manager 体验的相关组件

频道定义在 [[./source/channel.scm][source/channel.scm]]，版本锁定在 [[./source/channel.lock][source/channel.lock]]。

*版本锁定* 的相关文件 (=channel.lock=) 实际上利用了 channel 的参数中可以指定 *commit* 的特点，通过锁住 commit 来固定仓库

*** 2. System vs Home
Guix 有两层配置：

| 层级   | 管理范围                         | 对应命令    | 配置文件来源                     |
|--------+----------------------------------+-------------+----------------------------------|
| *System* | 内核、引导、系统服务、全局软件包 | =guix system= | [[source/configs/system-config.org]] |
| *Home*   | 用户软件包、dotfiles、用户服务   | =guix home=   | [[source/configs/home-config.org]]   |

*原则* ：能用 Home 解决就不用 System，保持系统层最小化。

*** 3. 利用 ~Org Mode~ 撰写配置
配置写在 =.org= 文件中，代码嵌入在 =src= 代码块里。这样做的好处：

- 配置和文档在一起
- 可以用 Emacs 的 =org-babel-tangle= 提取代码
- 阅读体验更好

本仓库使用 =maak= 自动处理 Org 文件的导出。

*** 关键入口
| 文件/目录                        | 用途                               | 修改频率                   |
|----------------------------------+------------------------------------+----------------------------|
| =source/information.scm=           | 定义用户名、Btrfs 子卷、持久化目录 | 初次使用时必改             |
| =source/channel.scm=               | 定义软件源频道                     | 需要添加第三方软件源时修改 |
| =source/configs/home-config.org=   | 用户软件包、服务、dotfiles 配置    | 经常                       |
| =source/configs/system-config.org= | 系统级配置                         | 较少                       |
| =dotfiles/immutable/*=             | 不可变配置（Guix Home 管理）       | 经常                       |
| =dotfiles/mutable/*=               | 可变配置（Stow 管理，独立调试）    | 经常                       |
| =maak.scm=                         | 任务定义                           | 需要添加自定义任务时修改   |

** 快速开始
*** 1. 安装系统（新机器）
#+begin_src sh
# 进入 Guix 安装环境后，克隆本仓库
git clone https://codeberg.org/BrokenShine/Guix-configs
cd ./Guix-configs

# 修改 source/information.scm 中的用户名等基本信息
# 然后初始化系统
guix shell maak -- maak init
#+end_src

*** 2. 日常更新（已安装系统）
#+begin_src sh
# 查看所有可用命令
maak --list

# 常用命令
maak system     # 应用系统配置（需要 sudo）
maak home       # 应用用户配置（Tangle → reconfigure → stow mutable dotfiles）
maak rebuild    # 先 system 后 home，并更新 locate 数据库

# 更新软件包频道
maak upgrade    # 更新系统中所有仓库到最新版本

# 验证配置（不实际应用）
MAAK_DRY_RUN=1 maak system   # 仅构建检查系统配置
MAAK_DRY_RUN=1 maak home     # 仅构建检查用户配置

# 括号平衡检查
maak check          # 检查 system + home 配置括号平衡
maak check-system   # 仅检查 system 配置
maak check-home     # 仅检查 home 配置
#+end_src

** 可复用组件指南
以下组件设计为可独立复用，你可以直接复制到自己的配置中：

*** ✅ 完全独立可复用
**** 1. 内核优化参数
([[./source/configs/system-config.org][source/configs/system-config.org]])

#+begin_src scheme
(kernel-arguments
  '("zswap.enabled=1"
    "zswap.compressor=zstd"
    "zswap.max_pool_percent=20"
    "zswap.zpool=zsmalloc"
    "net.ipv4.tcp_congestion_control=bbr"
    "net.ipv4.tcp_fastopen=3"
    "vm.vfs_cache_pressure=50"
    "kernel.sched_autogroup_enabled=1"))
#+end_src

*如何复用*：直接复制到 =operating-system= 的 =kernel-arguments= 字段。

**** 2. 静态配置文件 ([[./source/files/][source/files/]])

+ =nftables.conf= - 基于 nftables 的防火墙规则
+ =rounded.qss= - Qt 应用圆角样式
+ =skel/= - 骨架文件模板（git-credential-keepassxc 等）

*如何复用*：直接复制文件内容，或在自己的配置中使用 =(local-file "...")=
引用。

**** 3. Org 模式配置工作流
:PROPERTIES:
:CUSTOM_ID: 3-org-模式配置工作流
:END:
本仓库的核心创新：用 Org 文件管理配置，通过 =maak= 自动提取。

#+begin_src scheme
;; maak.scm 中的关键函数
(define (generate-config-from-org org-file)
  "使用 Emacs 的 org-babel-tangle 导出 Org 文件"
  ...)
#+end_src

*如何复用*：

1. 复制 =maak.scm=，并安装=maak=
2. 将 =.scm= 配置改写为 =.org= 格式
3. 用 =#+begin_src scheme :tangle ...= 包裹代码块

*** ⚠️ 需要适配后复用
**** 1. Btrfs + tmpfs 混合架构 ([[./source/information.scm][source/information.scm]])
实现「重启后根目录清空，但数据持久化」的不可变系统效果:
(具体需要持久化哪些目录，以及 btrfs 的相关结构，都需要做对应的修改)

#+begin_src scheme
;; 定义需要持久化的数据目录
(define %data-dirs
  '(".var"
    ".local/share/PrismLauncher"
    "Documents"
    "Downloads"
    "Pictures"
    ...))

;; 定义 Btrfs 子卷映射
(define %btrfs-subvolumes
  '(("SYSTEM/Guix/@data"   "/var/lib")
    ("SYSTEM/Guix/@gnu"    "/gnu")
    ("DATA/Home/Guix"      "/home")
    ...))
#+end_src

*如何复用*：参考 =information.scm= 中的 =%data-dirs= 和
=%btrfs-subvolumes= 定义，配合 =system-config.org= 中的 Btrfs 挂载配置。

**** 2. Dotfiles 组织方式 ([[./dotfiles/][dotfiles/]])
采用*两层结构*：

+ =immutable/= - 通过 Guix Home 的 =home-dotfiles-service-type= 管理，配置为只读，适合不需要频繁改动的软件
+ =mutable/= - 通过 [[https://www.gnu.org/software/stow/][GNU Stow]] 链接到用户目录，可独立修改调试，=maak home= 运行后自动执行 =stow-dotfiles= 重新链接

#+begin_example
dotfiles/
├── immutable/                          # Guix Home 管理（只读）
│   ├── desktop/.config/waybar/         # → ~/.config/waybar/
│   ├── terminal/.config/fish/          # → ~/.config/fish/
│   ├── system/.config/btop/            # → ~/.config/btop/
│   └── utilities/.config/tmux/         # → ~/.config/tmux/（侧边栏、窗口管理）
└── mutable/                            # Stow 管理（可独立调试）
    ├── emacs/.config/emacs/            # → ~/.config/emacs/
    ├── zed/.config/zed/                # → ~/.config/zed/
    └── tmux/.config/fish/conf.d/       # → ~/.config/fish/conf.d/
#+end_example

*如何复用*：

在 =home-config.org= 中声明 immutable 配置：

#+begin_src scheme
(service home-dotfiles-service-type
  (home-dotfiles-configuration
    (package `("desktop" "terminal" "system" ...))))
#+end_src

mutable 配置通过 ~maak home~ 中的 ~stow-dotfiles~ 函数自动链接到 =$HOME= 。

**** 3. tmux 配置 ([[./dotfiles/immutable/utilities/.config/tmux/][dotfiles/immutable/utilities/.config/tmux/]])
tmux 配置采用 Guile 驱动 + Bash 胶水的混合架构，核心设计如下：

+ *侧边栏导航树*：由浮动 pane 渲染的层级树（Session → Window → Pane），取代原生 tab bar。通过前缀键 =Ctrl+x B= 切换显示
+ *无插件架构*：所有功能通过外部脚本 + tmux hooks 实现，无 TPM 依赖
+ *信号驱动刷新*：操作触发 SIGUSR1 即时重绘，30 秒心跳兜底
+ *CJK 感知渲染*：侧栏文本使用 Guile 实现的中文宽度计算
+ *工作区快照*：支持将当前 session 的窗口布局保存/恢复为 JSON

目录结构：
#+begin_example
~/.config/tmux/
├── tmux.conf                       # 主配置
└── scripts/
    ├── sidebar-toggle              # 侧边栏生命周期管理（Bash）
    ├── sidebar-render.scm          # 数据采集 + 渲染引擎（Guile）
    ├── sidebar-data.scm            # 兼容 wrapper → sidebar-render.scm data
    ├── git-collector.scm           # 兼容 wrapper → sidebar-render.scm git
    ├── tmux-helpers.scm            # 共享工具模块（缓存、锁、CJK 宽度）
    ├── workspace.scm               # 工作区快照管理
    ├── session-selector            # 会话选择器（fzf）
    ├── window-jump                 # 窗口跳转（fzf）
    └── which-key                   # 快捷键帮助弹窗
#+end_example

*如何复用* ：复制 =tmux.conf= 和 =scripts/= 目录到自己的配置中。需要 tmux 3.3a+ 和 Guile 支持。Guix 用户可直接在 home 环境中添加 =guile= 包。
*注意* ：侧边栏依赖 foot 终端的行为，在其他终端下可能需要调整 *default-terminal* 和 *terminal-overrides* 设置。

** 配置分层策略
理解何时修改哪个文件：

*** 添加新软件包
1. *仅用户需要* → =source/configs/home-config.org=
   + 添加到 =packages= 字段
   + 示例：游戏、开发工具、媒体播放器

2. *系统级必需* → =source/configs/system-config.org=
   + 添加到 =packages= 字段
   + 示例：文件系统工具、系统监控、固件

*** 添加新服务
1. *用户服务*（如 swayidle、mako）→ =home-config.org=
2. *系统服务*（如 nftables、docker）→ =system-config.org=

*** 添加配置文件
:PROPERTIES:
:CUSTOM_ID: 添加配置文件
:END:
*优先放入 =dotfiles/=*，根据配置性质选择分层：

1. *不需要频繁修改* → 放入 =dotfiles/immutable/= ，通过 *home-config.org* 的 Guix Home 管理
2. *需要独立调试* → 放入 =dotfiles/mutable/= ，通过 *stow* 直接链接，修改后不需 =maak home=

#+begin_src sh
# 示例：添加 immutable 配置
mkdir -p dotfiles/immutable/app-name/.config/app-name
cp ~/app-config.toml dotfiles/immutable/app-name/.config/app-name/
# 然后在 home-config.org 中添加 app-name 到 packages 列表

# 示例：添加 mutable 配置
mkdir -p dotfiles/mutable/app-name/.config/app-name
cp ~/app-config.toml dotfiles/mutable/app-name/.config/app-name/
# 下次 maak home 运行时 stow-dotfiles 自动链接
#+end_src

*仅在需要动态生成时使用 source/files/* （如需要注入 Guix 软件包路径）。

** 常用任务
*** 修改用户名
编辑 =source/information.scm= ：

#+begin_src scheme
(define username "your-username")
#+end_src

然后重新运行 ~maak home~ 。

*** 添加 Btrfs 持久化目录
编辑 =source/information.scm= ：

#+begin_src scheme
(define %data-dirs
  '(... "your-new-dir"))
#+end_src

然后在 ~system-config.org~ 的 Btrfs 配置部分添加对应的 ~bind-mount~ 。

*** 添加新的 Channel
编辑 =source/channel.scm= ，添加频道定义，然后运行：

#+begin_src sh
maak upgrade
#+end_src

*** 自定义 =maak= 任务
编辑 ~maak.scm~ ，添加新函数：

#+begin_src scheme
(define (my-task)
  "我的自定义任务"
  (log-info "执行自定义任务")
  ($ `("echo" "Hello, Guix!")))
#+end_src

然后运行 ~maak my-task~ 。

** 故障排除
*** =maak home= 提示找不到目录
确保 =dotfiles/immutable/<name>= 目录存在，且与 =home-dotfiles-configuration= 中声明的 ~packages~ 一致。

*** 系统启动后数据丢失
检查 =source/information.scm= 中的 ~%data-dirs~ 和 ~Btrfs 子卷配置~ 是否正确，确保 =/data= 分区已正确挂载。

*** Org 导出失败
确保安装了 Emacs 且启用了 =org-babel-tangle=：

#+begin_src sh
emacs --batch -l org --eval "(require 'ob-tangle)"
#+end_src

*** 配置语法检查
修改 ~*.org~ 配置后，可以用 ~dry-run~ 模式验证语法是否正确（不会实际应用到系统）：

#+begin_src sh
MAAK_DRY_RUN=1 maak system   # 验证系统配置
MAAK_DRY_RUN=1 maak home     # 验证用户配置
#+end_src

~maak system~ 和 ~maak home~ 在执行前会自动运行括号平衡检查。也可以手动单独检查：

#+begin_src sh
maak check          # 检查所有配置
maak check-system   # 仅检查系统配置
#+end_src
