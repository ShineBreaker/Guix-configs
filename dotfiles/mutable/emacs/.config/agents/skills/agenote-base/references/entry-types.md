# ENTRY_TYPE 语义映射

ENTRY_TYPE 区分卡片的语义类型。agent 场景的映射如下：

## 三种 ENTRY_TYPE

### `note` — 主动记录的有用知识

**何时用**：agent 查询到的、值得长期保留的知识。

- 联网查到的技术方案细节
- 文档里的 API 用法、配置范例
- 环境信息、工具链约束

**默认 TYPE**：`workflow`

**body 章节**：事项内容 / 为什么值得长期保留 / 适用场景与例外 / 后续行动

**示例**：

```
agenote add --title "Guix 通道锁机制" --entry note
```

### `mistake` — 踩坑/被纠正

**何时用**：agent 犯了错或走了弯路。

- 被用户纠正的错误判断
- 误读了需求导致返工
- 用了错误的工具/方法

**默认 TYPE**：`debug`

**body 章节**：原始问题 / 用户纠错反馈 / 这次到底错在哪里 / 最终正确处理

**示例**：

```
agenote add --title "误改 tmp/ 产物" --entry mistake
```

### `ascended` — 多轮试错后的最优方案

**何时用**：agent 经历多次失败/重试后找到的正确做法。

- 前几轮方案都失败，最终找到可行方案
- 经过验证的最强方案，值得记录避免重试

**默认 TYPE**：`debug`

**body 章节**：前几轮失败的根因 / 检索过的知识源 / 核对过的真实文件或输出 / 最终采用的最强方案

**示例**：

```
agenote add --title "Emcc 启动卡死排查" --entry ascended
```

## 与人类场景的对照

| ENTRY_TYPE | 人类场景              | agent 场景                 |
| ---------- | --------------------- | -------------------------- |
| note       | 值得长期保留的笔记    | agent 查询到的有用知识     |
| mistake    | 踩坑/被纠正的经验教训 | agent 被纠正、走弯路       |
| ascended   | 多轮试错后的最优方案  | agent 多次重试后的正确做法 |
