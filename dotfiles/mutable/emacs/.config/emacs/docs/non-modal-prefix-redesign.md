# 非模态前缀键重构说明

## 这次改了什么

这次不再使用单一总前缀入口。

原先那种“先按一个总前缀，再进入所有功能树”的方式，已经不再适合这套配置。现在改成了更接近 Emacs 自身逻辑的多前缀分组：

- `C-x`：文件、缓冲区、窗口、标签
- `C-x s`：选择、多光标与选区扩展
- `C-x p`：项目
- `M-s`：搜索与替换
- `M-g`：跳转、错误、返回
- `C-c l`：代码 / LSP
- `C-c f`：格式化
- `C-c e`：编辑变换
- `C-c z`：折叠
- `C-c g`：Git
- `C-c t`：切换、工作区、工具面板
- `C-c a`：应用
- `C-c o`：Org 扩展
- `C-c h`：帮助

同时，前缀提示菜单也一起重做了：

- `which-key` 说明文本已集中汉化
- 底部弹窗改成中文友好的固定等宽网格
- 默认使用三列布局
- 宽屏可继续升到四列，较窄窗口则更适合两列

高频直达键也尽量贴近 Windows / 常见 IDE 习惯：

- `C-s`：保存
- `C-f`：当前缓冲区搜索
- `C-S-f`：项目全文搜索
- `C-p`：项目内找文件
- `C-S-b`：切换缓冲区

## 为什么这样改

原先的单一总前缀体系有两个问题：

1. 它要求你先记住“总入口”。
2. 它把很多本来就有自然归属的命令，硬塞进同一棵树里。

按 Emacs 本身的习惯，更自然的方式不是再找一个新的全局入口，而是让不同类型的命令回到不同前缀：

- 文件和窗口，本来就该靠近 `C-x`
- 项目，本来就该靠近 `C-x p`
- 搜索，本来就该靠近 `M-s`
- 跳转和错误，本来就该靠近 `M-g`
- 你自己扩展出来的能力，再放进 `C-c` 语义前缀

这样记忆方式会从“记住一棵大树”，变成“先想命令属于哪一类，再进对应前缀”。

## 为什么 `C-h` 不恢复帮助

这次保留了你的要求：

- `C-h` 继续是左移
- `C-j` 继续是下移
- `C-k` 继续是上移
- `C-l` 继续是右移

这意味着帮助前缀不能再依赖 Emacs 默认的 `C-h`。

所以帮助统一改成：

- `F1 ?`
- `C-c h ?`
- `C-c h B`：查看当前上下文绑定

这套安排的核心原则是：不牺牲你已经形成手感的光标移动逻辑，同时把帮助收口到一个稳定的替代前缀。

## 前缀分组说明

### 文件 / 缓冲区 / 窗口 / 标签

前缀：`C-x`

主要命令：

- `C-x C-f`：打开文件
- `C-x C-s`：保存
- `C-x C-w`：另存为
- `C-x C-r`：最近文件
- `C-x P`：打开 PDF
- `C-x b`：切换缓冲区
- `C-x B`：缓冲区列表
- `C-x k`：关闭当前缓冲区
- `C-x 2 / 3 / 0 / 1 / o`：窗口分割与切换
- `C-x t ...`：标签相关操作

### 项目

前缀：`C-x p`

主要命令：

- `C-x p p`：切换项目
- `C-x p f`：项目内找文件
- `C-x p s`：项目全文搜索

## 为什么 which-key 现在会更整齐

之前的问题不是“有没有汉化”，而是汉化之后中文宽度和 `which-key` 默认按字符数补空格的算法不一致，所以会出现：

- 已经全汉化，但列与列错位
- 第二列紧贴第一列
- 双列看起来像一长串文本，而不是分栏

这次改动做了两件事：

1. 说明文本单独收口到 `configs/i18n/which-key-descriptions.el`
2. `which-key` 的列补齐逻辑改成按显示宽度计算，并按可用区域等分列宽

现在的显示逻辑更接近常见软件里的面板式分栏：

- 每列按区域等分
- 列宽固定，不再贴着上一列内容跑
- 中文描述会按显示宽度截断，不再因为全角字符把后面一列挤歪

## 鼠标仍然是一等输入层

鼠标逻辑没有被降级成“备用方案”：

- `右键`：上下文菜单
- `C-<mouse-1>`：跳转定义
- `M-<mouse-1>`：添加多光标
- `S-<mouse-1>`：扩展选区
- `<mouse-8> / <mouse-9>`：后退 / 前进

## 现在的记忆方式

新的记忆方式不是：

- 先记一个总入口
- 再记整棵命令树

而是：

1. 先判断命令属于哪一类
2. 进入对应前缀
3. 再记那一类里的少量命令

例如：

- 文件 / 缓冲区 / 窗口：先想 `C-x`
- 项目：先想 `C-x p`
- 搜索：先想 `M-s`
- 跳转与错误：先想 `M-g`
- 代码能力：先想 `C-c l`
- Git：先想 `C-c g`
- 切换和工作区：先想 `C-c t`

这会比单一总前缀更接近 Emacs 本体，也更容易长期稳定扩展。
