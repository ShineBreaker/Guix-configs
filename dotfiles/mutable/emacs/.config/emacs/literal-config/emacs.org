* 启动顺序与模块依赖速查
本配置 =emacs.org= 由 8 个顶层域构成(末尾 =* init.el ends here= 为占位),每个域通过 13 处 =(require 'literal-...)= 把外置 =lisp/*.el= 模块挂到 init 序列。加载顺序敏感:任一处 require 顺序错位都可能引起 void-function / 注入点为 nil。

+ 路径常量由 =literal-bootstrap.el= 的 =defconst= 提供,模块 =defvar= 不覆盖已 bound 值
+ frame hook 实现由 =literal-frame.el= 的 =defun= 提供,经 =defvar= 注入点 setq 到 color-scheme / dashboard 模块
+ Dashboard 因依赖知识库/帮助/颜色方案模块,延后到 =* 系统工具与实验性= 域末尾加载;4 个回调在加载前 setq 注入

* 启动与基础设施
Emacs 冷启动的第一阶段:GC / 路径常量 / 外部命令缓存 / Frame 生命周期 / 键位辅助 / 包管理 / 基础行为。
本域代码必须最先执行——后续所有功能域(以及外置重型模块)都依赖这里定义的常量、命令缓存与 =literal/set-key= / =literal/add-frame-hook= 等基础原语。

** GC 与启动优化
临时调高 GC 阈值、清空 =file-name-handler-alist= 以加速启动,加载 =lisp/= 子目录,启动完成后恢复。

+ chemacs2 已把 =user-emacs-directory= 指向本目录(=literal-config/=)
+ =gc-cons-threshold= 启动期推到 =most-positive-fixnum= ,启动完成后由 =gcmh-mode= 重置为合理值(见 *基础行为设置* )
+ =file-name-handler-alist= 启动期清空(减少 =expand-file-name= 开销),启动完成后恢复
+ =lisp/= 子目录存放外置重型子系统(modeline / tab-line / dashboard / git / context-menu / which-key-data 等),启动期加入 =load-path=

#+begin_src emacs-lisp
;;; main.el --- literal-config 主配置(由 emacs.org tangle 生成) -*- lexical-binding: t; -*-

;; ═════════════════════════════════════════════════════════════════════════════
;; GC 启动期优化(启动完成后由 gcmh 重置)
;; ═════════════════════════════════════════════════════════════════════════════
(setq gc-cons-threshold most-positive-fixnum
      gc-cons-percentage 0.6)

;; file-name-handler-alist 启动期清空,启动后恢复
(defvar literal--file-name-handler-alist-original file-name-handler-alist
  "启动前的 file-name-handler-alist,启动完成后恢复。")
(setq file-name-handler-alist nil)

;; ═════════════════════════════════════════════════════════════════════════════
;; load-path 设置
;; ═════════════════════════════════════════════════════════════════════════════
(add-to-list 'load-path
             (expand-file-name "lisp" user-emacs-directory))

;; ═════════════════════════════════════════════════════════════════════════════
;; 启动完成后恢复 file-name-handler-alist
;; ═════════════════════════════════════════════════════════════════════════════
(add-hook 'emacs-startup-hook
          (lambda ()
            (setq file-name-handler-alist
                  literal--file-name-handler-alist-original)))

(message "[literal-config] 基础设施加载完成")
#+end_src

** 路径常量与外部命令缓存(外置 bootstrap)
集中定义所有路径常量(org 目录、guix profile、tree-sitter 库等),并在启动期一次性缓存所有外部命令路径。实现外置在 =lisp/literal-bootstrap.el= 。

+ 遵循 doom 范式: =defconst= 时调 =executable-find= ,后续只读变量,避免每次使用都搜 PATH
+ =early-init.el= 已把 =GUIX_PROFILE/bin= 和 =~/.local/bin= 加入 =exec-path= ,因此缓存能命中 =agenote= / =godot= / =dot= 等用户工具
+ 后续所有功能域与外置重型模块(dashboard / git / org-knowledge 等)都引用这里的 =literal:org-directory= 、 =literal:executable-*
+ =literal/call-process= :同步执行外部命令的通用封装,返回 =(STATUS . OUTPUT)=
+ 保留为独立 .el :因为多个重型模块(color-scheme / git)头部 =(require 'literal-bootstrap)= ,独立文件能保证 byte-compile 与运行时都不出错

** Frame 生命周期与 daemon(外置 frame)
daemon/client 架构下 per-frame 初始化(主题、字体、childframe、tab-line、dashboard 等)的统一注册入口。实现外置在 =lisp/literal-frame.el= 。

+ 替代旧配置的 =custom/register-daemon-frame-hook=
+ 在 daemon 模式下同时挂 =after-make-frame-functions= 与 =server-after-make-frame-hook= (覆盖 =emacs --fg-daemon= + =emacsclient= 两条路径)
+ 非 daemon 模式立即用当前 frame 调一次(standalone 冷启动)
+ 使用范式: =(literal/add-frame-hook #'my-setup-display)= ,而非手写 =add-hook=
+ =literal/daemon-runtime-p= 供延迟预热场景判断是否为可交互 daemon 会话
+ **插件化解耦**: =literal-color-scheme.el= / =literal-dashboard.el= 各自带 =defvar literal/add-frame-hook nil= 注入点(不 require frame),本块加载 frame 后把真实函数注入这两个变量。Lisp-2 双命名空间下 =defun= (函数)与模块 =defvar= (变量)同名共存,=funcall= 变量值即可调用

#+begin_src emacs-lisp
(require 'literal-frame)

;; 注入 frame hook 实现到自包含模块的 defvar 注入点
;; (color-scheme / dashboard 头部 (defvar literal/add-frame-hook nil))
;; Lisp-2: defun literal/add-frame-hook(函数)与 defvar(变量)同名共存
(setq literal/add-frame-hook    #'literal/add-frame-hook
      literal/remove-frame-hook #'literal/remove-frame-hook)
#+end_src

** 键位辅助函数
=literal/set-key= :一行完成全局键位绑定 + which-key 中文描述注册(替代旧 =custom/bind!= 宏)。

+ use-package 原生写法下,全局键位用 =keymap-global-set= ,但 which-key 中文描述需要单独调 =which-key-add-key-based-replacements= ,导致"绑定"与"描述"分离;本函数合并二者
+ which-key 尚未加载时,描述先收集到 =literal--pending-wk-descs= ,由 =literal/register-pending-wk-descs= 在 which-key 加载后一次性注册
+ **必须放在基础设施域**: =literal/register-pending-wk-descs= 会在 *界面与外观* 域的 which-key =:config= 里被调用,若晚于 which-key 加载会 void-function
+ Emacs 31 =keymap-global-set= 走严格 =key-valid-p= :修饰键必须写在 =<>= 外部( ="C-<tab>"= 而非 ="<C-tab>"= );本函数对含 =<= 的 KEY 自动转换格式
+ 本块同时 =provide 'literal-keys=

#+begin_src emacs-lisp
(defvar literal--pending-wk-descs nil
  "literal/set-key 收集的键位描述,which-key 加载后批量注册。")

(defun literal/set-key (key command &optional desc)
  "全局绑定 KEY 到 COMMAND,可选用 DESC 注册 which-key 中文描述。
KEY 与 DESC 均为字符串。DESC 为 nil 时只做绑定。
此函数替代旧配置的 `custom/bind!' 宏。

Emacs 31 的 `keymap-global-set' 走严格 `key-valid-p':
修饰键必须写在 <> 外部(\"C-<tab>\" 而非 \"<C-tab>\")。
本函数对包含 `<' 的 KEY 自动转换格式,兼容旧式写法。"
  (keymap-global-set key command)
  (when (and desc (stringp desc))
    (with-eval-after-load 'which-key
      (which-key-add-key-based-replacements key desc))
    (push (cons key desc) literal--pending-wk-descs)))

(defun literal/register-pending-wk-descs ()
  "把 `literal--pending-wk-descs' 注册到 which-key(一次性)。"
  (dolist (entry (nreverse literal--pending-wk-descs))
    (which-key-add-key-based-replacements (car entry) (cdr entry)))
  (setq literal--pending-wk-descs nil))

(provide 'literal-keys)
#+end_src

** 基础行为设置
GC 策略、文件备份、历史记录、自动保存、custom-file 位置等基础设置。

+ ~gcmh-mode~ :启动后接管 GC,空闲时自动 GC,编辑时不打断
+ ~no-littering~ :把各种包生成的文件统一放到 =var/= 和 =etc/=
+ ~savehist~ / =recentf= :minibuffer 历史与最近文件持久化

(use-package gcmh
  :demand t
  :custom
  (gcmh-idle-delay 'auto)
  (gcmh-auto-idle-delay-factor 10)
  (gcmh-high-cons-threshold (* 32 1024 1024)) ; 32MB(daemon 长运行优化)
  :config
  (gcmh-mode 1))

;; ═════════════════════════════════════════════════════════════════════════════
;; LSP 通信优化
;; ═════════════════════════════════════════════════════════════════════════════
(setq read-process-output-max (* 4 1024 1024) ; 4MB
      process-adaptive-read-buffering nil
      fast-but-imprecise-scrolling t)

;; ═════════════════════════════════════════════════════════════════════════════
;; 启动画面与基础体验
;; ═════════════════════════════════════════════════════════════════════════════
(setq inhibit-startup-screen t
      initial-scratch-message nil
      ring-bell-function 'ignore
      confirm-kill-emacs nil
      confirm-kill-processes nil)

;; ═════════════════════════════════════════════════════════════════════════════
;; no-littering + 统一管理缓存和配置文件
;; ═════════════════════════════════════════════════════════════════════════════
(use-package no-littering
  :demand t
  :config
  (setq backup-directory-alist
        `((".*" . ,(no-littering-expand-var-file-name "backup/"))))
  (setq auto-save-file-name-transforms
        `((".*" ,(no-littering-expand-var-file-name "auto-save/") t)))
  (setq custom-file (no-littering-expand-etc-file-name "custom.el"))
  (when (file-exists-p custom-file)
    (load custom-file t)))

;; ═════════════════════════════════════════════════════════════════════════════
;; auto-compile + 自动编译用户配置(不编译 Guix store)
;; ═════════════════════════════════════════════════════════════════════════════
(defun literal--auto-compile-root-directories ()
  "返回允许 auto-compile 处理的配置根目录。"
  (delete-dups
   (delq nil
         (list literal:emacs-dir
               literal:emacs-lisp-dir
               (file-truename literal:emacs-dir)
               (file-truename literal:emacs-lisp-dir)))))

(defun literal--auto-compile-file-p (file)
  "FILE 是否属于本配置并可自动编译。"
  (and (auto-compile-source-file-p file)
       (let ((expanded-file (expand-file-name file)))
         (catch 'matched
           (dolist (dir (literal--auto-compile-root-directories))
             (let ((root (file-name-as-directory (expand-file-name dir))))
               (when (string-prefix-p root expanded-file)
                 (throw 'matched t))))
           nil))))

(use-package auto-compile
  :demand t
  :config
  (setq auto-compile-use-mode-line nil)
  (setq auto-compile-predicate-function #'literal--auto-compile-file-p)
  (auto-compile-on-load-mode 1)
  (auto-compile-on-save-mode 1))

;; ═════════════════════════════════════════════════════════════════════════════

(use-package savehist
  :defer 0.5
  :config
  (savehist-mode 1))

(use-package recentf
  :defer 1
  :custom
  (recentf-max-saved-items 300)
  :config
  (recentf-mode 1))

;; ═════════════════════════════════════════════════════════════════════════════

(add-function :after after-focus-change-function

;; ═════════════════════════════════════════════════════════════════════════════

;; trusted-content(Emacs 30+,CVE-2024-53920 缓解)
(when (boundp 'trusted-content)

;; ═════════════════════════════════════════════════════════════════════════════
;; 行号显示(相对行号 + 自适应宽度)

(setq display-line-numbers-type 'relative)
(global-display-line-numbers-mode 1)

(defun literal/adjust-line-number-width ()
  "按当前 buffer 的总行数设置行号列宽度,避免大文件滚动时列宽跳变。"
  (when (and (boundp 'display-line-numbers-width)
             buffer-file-name
             global-display-line-numbers-mode)
    (let* ((max-line (line-number-at-pos (point-max) 'absolute))
           (digits (max 3 (length (number-to-string max-line)))))
      (setq display-line-numbers-width (1+ digits))
      (set-window-fringes (selected-window) 0 nil nil))))

(add-hook 'find-file-hook #'literal/adjust-line-number-width)

;; 在某些模式下禁用行号
(dolist (mode '(ghostel-mode term-mode eshell-mode shell-mode
                             dashboard-mode special-mode
                             agent-shell-mode))
  (add-hook (intern (concat (symbol-name mode) "-hook"))
            (lambda () (display-line-numbers-mode -1))))

;; ═════════════════════════════════════════════════════════════════════════════
;; 统一 per-frame 外观设置入口
;; ═════════════════════════════════════════════════════════════════════════════
(defun literal/apply-terminal-faces ()
  "为当前 frame 应用终端 ANSI 颜色(仅 TTY)。"
  (condition-case err
      (let ((tty (selected-frame)))
        (set-face-attribute 'highlight tty :background "brightblack" :foreground "white")
        (set-face-attribute 'region tty :background "brightblack" :foreground "white")
        (set-face-attribute 'hl-line tty :background "brightblack")
        (set-face-attribute 'show-paren-match tty :background "cyan" :foreground "black")
        (set-face-attribute 'show-paren-mismatch tty :background "red" :foreground "white")
        (set-face-attribute 'lazy-highlight tty :background "brightblack" :foreground "brightyellow")
        (set-face-attribute 'isearch tty :background "cyan" :foreground "black")
        (set-face-attribute 'line-number tty :foreground "brightblack")
        (set-face-attribute 'line-number-current-line tty :foreground "brightgreen")
        (set-face-attribute 'mode-line tty :background 'unspecified :foreground "white" :box nil)
        (set-face-attribute 'mode-line-inactive tty :background 'unspecified :foreground "brightblack" :box nil)
        (set-face-attribute 'help-key-binding tty :foreground "cyan")
        (set-face-attribute 'button tty :foreground "cyan"))
    (error
     (message "literal/apply-terminal-faces: %s" (error-message-string err)))))

(defun literal/apply-frame-appearance (&optional frame)
  "为 FRAME 应用所有 display-dependent 外观设置。
+ GUI: 字体高度、透明度、window-divider、tab-line face 重算
+ TTY: 终端 ANSI 颜色覆盖"
  (condition-case err
      (with-selected-frame (or frame (selected-frame))
        (let ((graphic-p (display-graphic-p)))
          (setq-default cursor-type (if graphic-p 'bar 'box))
          (if graphic-p

(defun literal/which-key--join-key (prefix key)
  "将 PREFIX 与 KEY 合并为 which-key 可用的键序列字符串。"
  (if (and prefix (> (length prefix) 0))
      (concat prefix " " key)
    key))

(defun literal/which-key--flatten-spec (prefix spec)
  "将 PREFIX 下的单个 SPEC 展开为 ((key . desc) ...) 列表。"
  (pcase spec
    (`(,key ,desc . ,children)
     (let ((full-key (literal/which-key--join-key prefix key)))
       (append (list (cons full-key desc))
               (literal/which-key--flatten-specs full-key children))))
    (`(,key . ,desc)
     (list (cons (literal/which-key--join-key prefix key) desc)))
    (_
     (error "无法识别的 which-key 描述项: %S" spec))))

(defun literal/which-key--flatten-specs (prefix specs)
  "将 PREFIX 下的 SPECS 展开为平铺的 ((key . desc) ...) 列表。"
  (apply #'append
         (mapcar (lambda (spec)
                   (literal/which-key--flatten-spec prefix spec))
                 specs)))

  (literal/which-key--flatten-specs nil literal:which-key-description-spec)
  "Which-key 全局中文描述映射表。")

  "避免重复写入 `which-key-replacement-alist'。")

  "将中文描述注册到 `which-key'。"
  (when (and (featurep 'which-key)
             (not literal:which-key-descriptions-applied))
    (dolist (entry literal:which-key-regexp-replacements)
      (setq which-key-replacement-alist
            (append which-key-replacement-alist (list entry))))
    (dolist (entry literal:which-key-replacements)
      (which-key-add-key-based-replacements (car entry) (cdr entry)))
    (dolist (mode-entry literal:which-key-major-mode-description-spec)
      (let ((mode (car mode-entry))
            (specs (cdr mode-entry)))
        (dolist (entry (literal/which-key--flatten-specs nil specs))
          (which-key-add-major-mode-key-based-replacements
            mode (car entry) (cdr entry)))))
    (setq literal:which-key-descriptions-applied t)))

** Which-key 本体与辅助包
加载 which-key 本体并配置显示参数;helpful 增强帮助系统;winner 窗口布局撤销 / 重做。

+ which-key 中文描述注册由 *界面与外观 → 右键菜单与 Which-key 中文化* 域处理( =literal/which-key-apply-descriptions= + =literal/register-pending-wk-descs= )
+ *中文字符对齐修复*: which-key 原生按 =length= (字符数)计算描述补齐宽度,中文双宽字符会让列错位。这里用 =advice-add :override= 替换 =which-key--pad-column= ,改为基于 =string-width= (显示列宽)计算,底部多列布局下固定为等宽分栏(默认 =literal:which-key-grid-columns 3= 列)
+ =which-key-idle-delay 0.3= :延迟显示,避免干扰快速操作
+ =winner-dont-bind-my-keys t= :避免 winner 抢占 =C-c ←= / =C-c →= (本配置在 =C-c w= 体系下手动绑定 winner-undo / redo)

;; ═════════════════════════════════════════════════════════════════════════════
;; Which-key(键位提示)
;; ═════════════════════════════════════════════════════════════════════════════

;; ── 中文字符对齐修复 ──────────────────────────────────────────────────────
;; which-key 原生按 `length'(字符数)计算描述补齐宽度,中文是双宽字符会导致列错位。
;; 这里统一改成基于 `string-width'(显示列宽)计算,并在底部多列布局下固定为等宽分栏。
;; 中文菜单默认三列网格;宽屏可升到 4,窄窗口则更适合降到 2。
(defconst literal:which-key-grid-columns 3
  "Which-key 中文菜单默认使用的固定网格列数。")

(defun literal/which-key--fixed-grid-layout-p ()
  "判断当前是否启用固定多列 which-key 网格布局。"
  (and (boundp 'which-key-max-display-columns)
       (integerp which-key-max-display-columns)
       (> which-key-max-display-columns 1)
       (boundp 'which-key-popup-type)
       (eq which-key-popup-type 'side-window)
       (boundp 'which-key-side-window-location)
       (eq which-key-side-window-location 'bottom)))

(defun literal/which-key--target-grid-column-width (available-width)
  "根据 AVAILABLE-WIDTH 计算固定网格布局下每列的目标宽度。"
  (if (literal/which-key--fixed-grid-layout-p)
      (let* ((columns which-key-max-display-columns)
             (gaps (max 0 (1- columns))))
        (/ (max 1 (- available-width gaps)) columns))
    available-width))

(defun literal/which-key--pad-column-fixed-width (col-keys available-width)
  "为 COL-KEYS 生成更适合中文的 which-key 列布局。
AVAILABLE-WIDTH 为当前 which-key 可用宽度。"
  (let* ((grid-layout-p (literal/which-key--fixed-grid-layout-p))
         (col-key-width (+ which-key-add-column-padding
                           (which-key--max-len col-keys 0)))
         (col-sep-width (which-key--max-len col-keys 1))
         (target-col-width (literal/which-key--target-grid-column-width available-width))
         (desc-space (max 1 (- target-col-width col-key-width col-sep-width)))
         (col-desc-width (if grid-layout-p
                             desc-space
                           (max which-key-min-column-description-width desc-space)))
         (col-width (+ col-key-width col-sep-width col-desc-width))
         (col-format (concat "%" (int-to-string col-key-width) "s%s%s")))
    (cons col-width
          (mapcar
           (pcase-lambda (`(,key ,sep ,desc ,_doc))
             (let* ((display-desc (truncate-string-to-width
                                   desc col-desc-width 0 nil which-key-ellipsis))
                    (padding (max 0 (- col-desc-width (string-width display-desc)))))
               (concat
                (format col-format key sep display-desc)
                (make-string padding ?\s))))
           col-keys))))

        ;; 中文条目在终端里更适合稳定的等宽分栏布局。

        which-key-side-window-max-height 0.38
        which-key-sort-order #'which-key-prefix-then-key-order
        which-key-C-h-map-prompt
        " \\<which-key-C-h-map>\\[which-key-show-next-page-cycle]/\\[which-key-show-previous-page-cycle] 翻页,\\[which-key-show-standard-help] 帮助,\\[which-key-abort] 退出")
  ;; 用基于 `string-width' 的中文友好版覆盖 which-key 原生 `which-key--pad-column'
  ;; (原生版按 `length' 计算补齐,中文双宽字符会导致列错位)。
  (advice-add 'which-key--pad-column :override #'literal/which-key--pad-column-fixed-width))

org src 块(§「代码操作辅助函数」,行 1736–1749)的三个 defun 是外置化 =literal-tab-line.el= 之前的简陋封装:
