维护约定:

1. 功能章节使用稳定 =CUSTOM_ID= ;改名标题时不要改 ID。
2. 大模块拆成 =module/section= 命名片段,章节末尾只有一个组装块写入 =main.el= 。
3. 改动局限在一个功能子树;确有跨域接口变更时同步更新上方依赖表。
4. 新包仍需同步 Guix 的 =home-emacs-packages= ;仅重排/精简现有代码不需要改包清单。
5. 提交前至少运行 =scripts/configctl check= 与 =scripts/configctl load= 。

** GC 与启动优化
:PROPERTIES:
:CUSTOM_ID: startup-gc
:END:

GC 启动期优化(启动完成后由 gcmh 重置)

+ 与 early-init.el 的同设置重复,但保留作为 main.el 自身的防御性优化——若有人绕过 early-init.el 直接 batch-load main.el(如 =configctl load= ),GC threshold 仍需在 main.el 入口推高。
+ early-init.el 的 =file-name-handler-alist = nil= 已足够覆盖正常启动路径,这里不再重复设置。
+ 临时调高 GC 阈值以加速启动,启动完成后由 =gcmh-mode= 接管。

+ =user-emacs-directory= 由当前单 profile 直接指向本目录
+ =gc-cons-threshold= 启动期推到 =most-positive-fixnum= ,启动完成后由 =gcmh-mode= 重置为合理值(见 *基础行为设置* )
+ =gc-cons-percentage= 启动期推到 0.6(减少启动期 GC),启动完成后显式恢复到 0.1——GCMH 只动态调整 =gc-cons-threshold= ,不管 =gc-cons-percentage= ,故必须显式恢复,否则 GCMH 接管后 percentage 仍停留在 0.6。
+ 保留 Emacs 31 默认的 =file-name-handler-alist= 与 bidi 行为。daemon 冷启动不是目标，不能用未测量的优化破坏 TRAMP、压缩文件或 RTL 文本。

#+begin_src emacs-lisp
;;; main.el --- literal-config 主配置(由 emacs.org tangle 生成) -*- lexical-binding: t; -*-

;; SPDX-FileCopyrightText: 2026 BrokenShine <xchai404@gmail.com>
;; SPDX-License-Identifier: MIT

(setq gc-cons-threshold most-positive-fixnum
      gc-cons-percentage 0.6)

(add-hook 'emacs-startup-hook
          (lambda ()
            (setq gc-cons-percentage 0.1)))

(message "[literal-config] 基础设施加载完成")
#+end_src

** 共享进程辅助
:PROPERTIES:
:CUSTOM_ID: process-helper
:END:

统一的同步进程入口与 agenote CLI 适配层。

+ =literal/call-process= :通用同步子进程(返回 =(STATUS . OUTPUT)=)
+ =literal/agenote-call= :agenote CLI 唯一入口,domain 必填,返回 plist
+ =literal/agenote-call-async= :长任务异步执行(reindex/lint/deduplicate/commit/viz)

agenote CLI 支持 =--domain human|agenote=,默认 agenote(与 MCP server 对齐)。

Emacs 端必须为每次调用显式选择 domain —— 不依赖默认值,避免人类域操作静默打到 agent 子域。

**不缓存 Guix store 绝对路径* *:
+ daemon 长驻跨 Guix profile 更新时,缓存的 =/gnu/store/.../bin/agenote= 会指向旧 generation。
+ =literal/agenote-call= 内部按需 =(executable-find "agenote")=,让每次调用都解析当前 PATH。

#+begin_src emacs-lisp :noweb-ref helpers/call-process :tangle no
(defun literal/call-process (command &rest args)
  "同步执行 COMMAND 与 ARGS,返回 (STATUS . OUTPUT)。"
  (with-temp-buffer
    (cons (or (apply #'call-process command nil t nil (remq nil args)) -1)
          (string-trim (buffer-string)))))

(defconst literal:agenote-valid-domains '(human agenote)
  "agenote CLI 支持的 domain 取值。与 ~agenote --help~ 对齐。")

(defun literal/agenote--resolve-executable ()
  "按需解析 agenote 命令路径。返回可执行路径字符串或 nil。
不缓存绝对 store 路径:daemon 跨 Guix generation 后能找到新 profile 的 agenote。
实际调用始终走本函数。"
  (executable-find "agenote"))

(defun literal/agenote--ensure-domain (domain)
  "校验 DOMAIN,返回小写 symbol。无效值 fail-fast。"
  (let ((d (if (stringp domain) (intern (downcase domain)) domain)))
    (unless (memq d literal:agenote-valid-domains)
      (error "literal/agenote-call: domain must be %s, got %S"
             (mapconcat #'symbol-name literal:agenote-valid-domains "|")
             domain))
    d))

(cl-defun literal/agenote-call (domain command &rest args)
  "同步调用 agenote CLI,DOMAIN 必填(human 或 agenote),COMMAND 子命令字符串,
ARGS 附加参数。

返回 plist:`(:status :stdout :stderr :command :domain)'。
- :status  进程退出码(-1 表示无法启动)
- :stdout  stdout 字符串(trimmed)
- :stderr  stderr 字符串(trimmed)
- :command  完整 argv 列表(便于诊断)
- :domain  实际使用的 domain symbol

用法示例:
  (literal/agenote-call 'human \"list\" \"--all\")
  (literal/agenote-call 'agenote \"search\" \"emacs\" \"--limit\" \"10\")
  (plist-get (literal/agenote-call 'human \"stats\") :stdout)"
  (let* ((dom (literal/agenote--ensure-domain domain))
         (exe (literal/agenote--resolve-executable))
         (argv (delq nil (append (list exe "--domain" (symbol-name dom) command) args))))
    (if (not exe)
        (list :status -1 :stdout "" :stderr "agenote executable not found"
              :command argv :domain dom)
      (let ((stdout-buffer (generate-new-buffer " *agenote-stdout*"))
            (stderr-file (make-temp-file "agenote-stderr"))
            status)
        (unwind-protect
            (progn
              ;; call-process 签名:(PROGRAM INFILE BUFFER DISPLAY &rest ARGS)
              ;; BUFFER 为 list 形式时:(STDOUT-DEST STDERR-DEST),STDERR-DEST
              ;; 必须是文件名(string),不是 buffer object。我们用临时文件
              ;; 捕获 stderr,再读回字符串。
              ;; PROGRAM 必须是 string;argv 是 (exe "--domain" "..." cmd args...),
              ;; 拆出 exe 作为 PROGRAM,其余作为 ARGS。
              (setq status
                    (apply #'call-process
                           (car argv) nil (list stdout-buffer stderr-file) nil
                           (cdr argv)))
              (list :status (or status -1)
                    :stdout (string-trim
                             (with-current-buffer stdout-buffer (buffer-string)))
                    :stderr (string-trim
                             (with-temp-buffer
                               (insert-file-contents stderr-file)
                               (buffer-string)))
                    :command argv
                    :domain dom))
          (when (buffer-live-p stdout-buffer) (kill-buffer stdout-buffer))
          (when (file-exists-p stderr-file) (delete-file stderr-file)))))))

(cl-defun literal/agenote-call-async (domain command &rest args)
  "异步调用 agenote CLI,参数同 `literal/agenote-call'。
长任务(reindex / lint / deduplicate / commit / viz)应使用本函数,避免阻塞
Emacs。结果写入 `*agenote-<command>*' buffer(special-mode,`g' 重跑,`q' 退出)。

返回启动的 process 对象;同步路径请用 `literal/agenote-call'。"
  (let* ((dom (literal/agenote--ensure-domain domain))
         (exe (literal/agenote--resolve-executable))
         (argv (delq nil (append (list exe "--domain" (symbol-name dom) command) args)))
         (buf-name (format "*agenote-%s*" command))
         (buffer (get-buffer-create buf-name)))
    (if (not exe)
        (progn
          (with-current-buffer buffer
            (let ((inhibit-read-only t))
              (erase-buffer)
              (insert "agenote executable not found\n")))
          (display-buffer buffer)
          nil)
      (with-current-buffer buffer
        (let ((inhibit-read-only t))
          (erase-buffer)
          (insert (format "$ %s\n\n" (string-join argv " ")))
          (special-mode)
          ;; `g' re-runs the same command.
          (let ((mode-map literal/agenote-async-mode-map))
            (use-local-map (or mode-map (make-sparse-keymap))))
          (define-key (current-local-map) "g"
                      (lambda () (interactive)
                        (apply #'literal/agenote-call-async dom command args)))))
      (let ((proc (make-process
                   :name (format "agenote %s" command)
                   :buffer buffer
                   :command argv
                   :connection-type 'pipe
                   :sentinel (lambda (proc event)
                               (let ((buf (process-buffer proc)))
                                 (when (buffer-live-p buf)
                                   (with-current-buffer buf
                                     (let ((inhibit-read-only t))
                                       (goto-char (point-max))
                                       (insert
                                        (format "\n[process %s]\n"
                                                (string-trim event)))))))))))
        (display-buffer buffer)
        proc))))

;; minimal keymap for async result buffers (defined here so async functions can
;; reference it before the dashboard/knowledge sections are loaded)
(defvar literal/agenote-async-mode-map
  (let ((m (make-sparse-keymap)))
    (set-keymap-parent m special-mode-map)
    m)
  "Keymap for `*agenote-<command>*' async result buffers.")

(defun literal/agenote-call-string (domain command &rest args)
  "同步调 agenote,返回 stdout 字符串(向后兼容 `literal/call-process' 调用)。
DOMAIN/COMMAND/ARGS 同 `literal/agenote-call'。失败返回空串。
新代码应直接用 `literal/agenote-call';本 wrapper 仅为兼容旧调用点保留。"
  (plist-get (apply #'literal/agenote-call domain command args) :stdout))

** Frame 生命周期与 daemon
:PROPERTIES:
:CUSTOM_ID: frame
:END:

daemon/client 架构下的两阶段 frame 生命周期：created 只处理 display 初始化，server-ready 处理依赖最终启动 buffer 的行为。每个 initializer 通过 frame parameter 保证成功执行一次。

#+begin_src emacs-lisp
(defvar literal--frame-created-initializers nil)
(defvar literal--server-ready-initializers nil)

(defun literal--run-frame-initializers (frame phase functions)
  "在 FRAME 运行 PHASE 对应的 FUNCTIONS，每个函数成功执行一次。"
  (when (frame-live-p frame)
    (let* ((parameter (pcase phase
                        ('created 'literal-frame-created-done)
                        ('server-ready 'literal-frame-server-ready-done)))
           (done (frame-parameter frame parameter)))
      (dolist (function functions)
        (unless (memq function done)
          (condition-case err
              (progn
                (funcall function frame)
                (push function done)
                (set-frame-parameter frame parameter done))
            (error
             (message "[frame:%s] %s 失败: %s"
                      phase function (error-message-string err)))))))))

(defun literal--dispatch-frame-created (frame)
  "运行 FRAME 的 created initializer。"
  (literal--run-frame-initializers
   frame 'created literal--frame-created-initializers))

(defun literal--dispatch-server-ready (&optional frame)
  "运行 FRAME 的 server-ready initializer。"
  (literal--run-frame-initializers
   (or frame (selected-frame)) 'server-ready literal--server-ready-initializers))

(defun literal/add-frame-created-hook (function)
  "把 FUNCTION 注册到 frame-created 阶段。"
  (unless (memq function literal--frame-created-initializers)
    (setq literal--frame-created-initializers
          (append literal--frame-created-initializers (list function))))
  (add-hook 'after-make-frame-functions #'literal--dispatch-frame-created)
  (unless (daemonp)
    (literal--dispatch-frame-created (selected-frame))))

(defun literal/add-server-ready-hook (function)
  "把 FUNCTION 注册到 server-ready 阶段。"
  (unless (memq function literal--server-ready-initializers)
    (setq literal--server-ready-initializers
          (append literal--server-ready-initializers (list function))))
  (if (daemonp)
      (add-hook 'server-after-make-frame-hook #'literal--dispatch-server-ready)
    (if after-init-time
        (literal--dispatch-server-ready (selected-frame))
      (add-hook 'emacs-startup-hook #'literal--dispatch-server-ready))))

(defun literal/daemon-runtime-p ()
  "返回当前是否为可交互的 daemon 会话（用于延迟预热等场景）。"
  (and (daemonp) (not noninteractive)))
#+end_src

** 键位声明
:PROPERTIES:
:CUSTOM_ID: key-helper
:END:

=literal:binding-spec= 是本地键位、Which-key、帮助与 Dashboard 的唯一声明源。每条声明在绑定真实 keymap 的同时登记描述；第三方既有 keymap 的翻译继续留在 =data/which-key-zh.el= 。

#+begin_src emacs-lisp
(defvar literal:binding-spec nil
  "本地声明式键位表。")

(defvar literal--pending-wk-descs nil
  "Which-key 加载前暂存的全局键位描述。")

(defun literal/binding--record (entry)
  "把 ENTRY 追加到 binding spec，并拒绝重复 key/mode。"
  (let ((key (plist-get entry :key))
        (mode (or (plist-get entry :mode) 'global)))
    (when (seq-find
           (lambda (existing)
             (and (equal key (plist-get existing :key))
                  (eq mode (or (plist-get existing :mode) 'global))))
           literal:binding-spec)
      (error "重复键位声明: %s (%s)" key mode))
    (setq literal:binding-spec
          (append literal:binding-spec (list entry)))))

(defun literal/binding--register-which-key (key description &optional mode)
  "为 KEY 注册 DESCRIPTION；MODE 非 nil 时注册 major-mode 描述。"
  (cond
   ((and mode (featurep 'which-key))
    (which-key-add-major-mode-key-based-replacements mode key description))
   ((featurep 'which-key)
    (which-key-add-key-based-replacements key description))
   ((not mode)
    (push (cons key description) literal--pending-wk-descs))))

(defun literal/bind (key target description group &rest properties)
  "声明并绑定全局 KEY 到 TARGET。
DESCRIPTION、GROUP 与 PROPERTIES 同时供 Which-key、帮助和 Dashboard 使用。"
  (let ((entry (append (list :key key
                             :command target
                             :description description
                             :group group)
                       properties)))
    (literal/binding--record entry)
    (unless (plist-get entry :prefix)
      (keymap-global-set key target))
    (literal/binding--register-which-key key description)
    entry))

(defun literal/bind-local (feature mode map key command description group)
  "声明 FEATURE 中 MODE/MAP 的局部 KEY。"
  (let ((entry (list :key key
                     :command command
                     :description description
                     :group group
                     :mode mode
                     :map map)))
    (literal/binding--record entry)
    (eval-after-load feature
      (lambda ()
        (keymap-set (symbol-value map) key command)
        (literal/binding--register-which-key key description mode)))
    entry))

(defun literal/declare-binding-prefix (key description)
  "声明 Dashboard 展示的前缀 KEY。"
  (literal/bind key nil description nil :prefix t :dashboard t))

(defun literal/register-pending-wk-descs ()
  "把暂存的全局键位描述注册到 Which-key。"
  (dolist (entry (nreverse literal--pending-wk-descs))
    (which-key-add-key-based-replacements (car entry) (cdr entry)))
  (setq literal--pending-wk-descs nil))

                    (plist-get entry :description))
              (alist-get group table nil nil #'equal))))

     (lambda (group)

     order)))

(defun literal/context-menu--translate-string (string)
  "翻译 STRING；未登记时返回原字符串。"
  (let* ((plain (substring-no-properties string))
         (translation (cdr (assoc plain literal:context-menu-label-translations))))
    (if (not translation)
        string
      (if (> (length string) 0)
          (apply #'propertize translation (text-properties-at 0 string))
        translation))))

(defun literal/context-menu--translate-object (object)
  "递归翻译 OBJECT 中的字符串。"
  (cond
   ((stringp object)
    (literal/context-menu--translate-string object))
   ((consp object)
    (mapcar #'literal/context-menu--translate-object object))
   (t object)))

(defun literal/context-menu--translate-menu-item (item)
  "翻译菜单项 ITEM 的标题和子菜单。"
  (when (and (consp item) (eq (car item) 'menu-item))
    (let ((label-cell (cdr item))
          (binding (nth 2 item))
          (properties (nthcdr 3 item)))
      (when label-cell
        (setcar label-cell
                (literal/context-menu--translate-object (car label-cell))))
      (when (keymapp binding)
        (literal/context-menu--translate-keymap binding))
      (while (consp properties)
        (when (and (eq (car properties) :help)
                   (consp (cdr properties)))
          (setcar (cdr properties)
                  (literal/context-menu--translate-object (cadr properties))))
        (setq properties (cddr properties))))))

(defun literal/context-menu--translate-keymap (keymap)
  "递归翻译 KEYMAP 内的菜单标题。"
  (when (keymapp keymap)
    (when (and (consp keymap)
               (eq (car-safe keymap) 'keymap)
               (stringp (cadr keymap)))
      (setcar (cdr keymap)
              (literal/context-menu--translate-string (cadr keymap))))
    (map-keymap
     (lambda (_event binding)
       (cond
        ((and (consp binding) (eq (car binding) 'menu-item))
         (literal/context-menu--translate-menu-item binding))
        ((keymapp binding)
         (literal/context-menu--translate-keymap binding))))
     keymap))
  keymap)

(defun literal/context-menu-translate (menu)
  "翻译 `context-menu-map' 返回的 MENU。"
  (literal/context-menu--translate-keymap menu))

(when (fboundp 'context-menu-map)
  (unless (advice-member-p #'literal/context-menu-translate #'context-menu-map)
    (advice-add 'context-menu-map :filter-return #'literal/context-menu-translate)))

(with-eval-after-load 'magit-section
  (unless (advice-member-p #'literal/context-menu--translate-string

** Tab-line
:PROPERTIES:
:CUSTOM_ID: tab-line
:END:

基于内置 =tab-line= 的 per-frame 文件标签,隔离多个 emacsclient frame 的缓冲区序列。

+ 禁用 frame 级 =tab-bar=,改用 window-local =tab-line= 作为文件标签栏——每个 window 一条标签栏,避免与 [[#frame][Frame 生命周期]] 的 per-frame 隔离冲突。
+ 只有关联文件的 buffer 才出现在标签栏(=literal/tab-visible-p= 白名单),临时 buffer 与 minibuffer 自动排除。

*** Per-frame 缓冲区跟踪

每个 frame 维护一个有序的文件 buffer 列表,存储在 frame parameter =literal--frame-tab-buffers= 中。这是多 emacsclient frame 隔离的核心:不同 frame 的标签序列完全独立。

#+attr_org: :width 60
| 状态变更      | 触发 hook                              | 行为                                     |
|---------------+----------------------------------------+------------------------------------------|
| 缓冲区可见    | =window-buffer-change-functions=         | 唯一注册路径:追加到 frame 列表尾部(去重) |
| 缓冲区被 kill | =kill-buffer-hook=                       | 从所有 frame 的列表移除                  |
| frame 被删除  | =delete-frame-functions=                 | 丢弃该 frame 的标签状态                  |
| 项目切换      | =project-after-switch-project-functions= | 清空当前 frame 列表(见 [[#project-navigation][项目与导航]])       |

注册顺序即标签排列顺序(append-only),保证用户切换的稳定心智模型。=window-buffer-change-functions= 是唯一的注册入口,不再依赖 =find-file-hook= / =server-switch-hook= 等重复触发源。

#+begin_src emacs-lisp :noweb-ref appearance/tab-line-tracking :tangle no
(defun literal--tabs-get-frame-buffer-list (&optional frame)
  "返回 FRAME 的标签缓冲区列表。"
  (frame-parameter (or frame (selected-frame)) 'literal--frame-tab-buffers))

(defun literal--tabs-set-frame-buffer-list (buffers &optional frame)
  "设置 FRAME 的标签缓冲区列表为 BUFFERS。"
  (set-frame-parameter (or frame (selected-frame)) 'literal--frame-tab-buffers buffers))

(defun literal--tabs-register-buffer (buffer &optional frame)
  "将 BUFFER 注册到 FRAME 的标签列表尾部(去重)。"
  (let* ((target-frame (or frame (selected-frame)))
         (buffers (or (literal--tabs-get-frame-buffer-list target-frame) '())))
    (unless (memq buffer buffers)
      (literal--tabs-set-frame-buffer-list
       (append buffers (list buffer))
       target-frame))))

(defun literal--tabs-unregister-buffer (buffer &optional frame)
  "从 FRAME 的标签列表中移除 BUFFER。"
  (let* ((target-frame (or frame (selected-frame)))
         (buffers (or (literal--tabs-get-frame-buffer-list target-frame) '())))
    (when (memq buffer buffers)
      (literal--tabs-set-frame-buffer-list
       (delq buffer buffers)
       target-frame))))

(defun literal--tabs-on-window-buffer-change (frame-or-window)
  "缓冲区变更时,把新缓冲区注册到对应 frame。
FRAME-OR-WINDOW 可以是 frame(全局 hook)或 window(buffer-local hook)。
这是标签列表的唯一注册入口,不依赖 find-file-hook / server-switch-hook。"
  (cond
   ((frame-live-p frame-or-window)
    (walk-windows
     (lambda (window)
       (when (eq (window-frame window) frame-or-window)
         (let ((buffer (window-buffer window)))
           (when (and (buffer-live-p buffer)
                      (literal/tab-visible-p buffer)
                      (not (minibufferp buffer)))
             (literal--tabs-register-buffer buffer frame-or-window)))))
     'no-mini frame-or-window))
   ((window-live-p frame-or-window)
    (let ((buffer (window-buffer frame-or-window))
          (frame (window-frame frame-or-window)))
      (when (and (buffer-live-p buffer)
                 (literal/tab-visible-p buffer)
                 (not (minibufferp buffer)))
        (literal--tabs-register-buffer buffer frame))))))

(defun literal--tabs-on-kill-buffer ()
  "缓冲区被关闭时,从所有 frame 的标签列表中移除。"
  (let ((buffer (current-buffer)))
    (dolist (frame (frame-list))
      (when (frame-live-p frame)
        (let ((buffers (literal--tabs-get-frame-buffer-list frame)))
          (when (memq buffer buffers)
            (literal--tabs-set-frame-buffer-list (delq buffer buffers) frame)))))))

(defun literal/tabs-clear-for-frame (&optional frame)
  "清空 FRAME 的标签缓冲区列表(项目切换时调用)。
只清 frame parameter,不重建;后续 window-buffer-change 自然填充。"
  (interactive)
  (let ((target-frame (or frame (selected-frame))))
    (literal--tabs-set-frame-buffer-list nil target-frame)
    (walk-windows
     (lambda (window)
       (when (eq (window-frame window) target-frame)
         (set-window-parameter window 'tab-line-cache nil)))
     'no-mini target-frame)
    (when (called-interactively-p 'interactive)
      (message "已清空当前 frame 的标签列表"))))

(defun literal--tabs-on-frame-deletion (frame)
  "FRAME 删除时清理标签状态。"
  (literal--tabs-set-frame-buffer-list nil frame))

(defun literal--tabs-frame-buffers (&optional window)
  "返回 WINDOW 所在 frame 的存活 + 可见缓冲区列表。"
  (let* ((target-window (literal--tabs-target-window window))
         (target-frame (window-frame target-window))
         (frame-buffers (literal--tabs-get-frame-buffer-list target-frame)))
    (seq-filter (lambda (buf)
                  (and (buffer-live-p buf)
                       (literal/tab-visible-p buf)))
                (or frame-buffers '()))))
#+end_src

*** 按钮与 Face

(defun literal/tabs-close-buffer (&optional window)
  "关闭当前标签(bury + unregister,不 kill buffer)。
键盘与鼠标关闭语义一致;原鼠标分支缺 fallback 的不对称已修正。"
  (interactive)
  (let* ((target-window (literal--tabs-target-window window))
         (buffer (window-buffer target-window))
         (target-frame (window-frame target-window)))
    (literal--tabs-unregister-buffer buffer target-frame)
    (if-let* ((next (seq-find (lambda (b)
                                (and (buffer-live-p b)
                                     (literal/tab-visible-p b)
                                     (not (eq b buffer))))
                              (literal--tabs-get-frame-buffer-list target-frame))))
        (literal--tabs-switch-to-buffer next target-window)
      (with-selected-window target-window
        (bury-buffer buffer)
        (let ((fallback (seq-find (lambda (b)
                                    (and (buffer-live-p b)
                                         (not (eq b buffer))))
                                  (buffer-list target-frame))))
          (when fallback
            (switch-to-buffer fallback)))
        (tab-line-force-update t)))))

  (let ((buffer (if (bufferp tab) tab (current-buffer))))
    (when (buffer-live-p buffer)
      (save-window-excursion
        (select-window (literal--tabs-target-window))
        (switch-to-buffer buffer)
        (literal/tabs-close-buffer)))))

(cl-defun literal/register-side-window (regex side size &key slot width)
  "把 REGEX 匹配的 buffer 路由到 SIDE 边的 side-window，占用 SIZE 比例。

REGEX : buffer-name 匹配字符串正则
SIDE  : bottom / right / left / top（符号）
SIZE  : 0~1 浮点比例
SLOT  : 默认 0（同 side 多窗口的堆叠顺序）
WIDTH : 非 nil 强制按 window-width；nil（默认）时按 SIDE 自动推导
        （left/right → window-width，top/bottom → window-height）。

规则追加到 `display-buffer-alist' 末尾。顺序敏感：更具体的规则应先注册。"
  (let* ((slot (or slot 0))
         (use-width (or width (memq side '(left right))))
         (window-dim (if use-width 'window-width 'window-height)))
    (add-to-list 'display-buffer-alist
                 `(,regex
                   (display-buffer-in-side-window)
                   (side . ,side)
                   (slot . ,slot)
                   (,window-dim . ,size))
                 t)))

(defun literal/display-buffer-in-main-window (buffer &optional _alist)
  "把 BUFFER 显示在「主编辑窗口」并选中。
主编辑窗口 = 当前 frame 内最靠上（y 坐标最小）的非 side-window 窗口。
保证无论光标当前在哪个窗口（如底部 magit），COMMIT_EDITMSG
都稳定落到上方编辑窗口，与「上编辑 + 下 magit」布局一致。

若无候选（如 frame 只剩 side-window），退化为 `display-buffer-pop-up-window'。

同时用作 `display-buffer-alist' 的 action 函数（参数 ALIST 被忽略）和
`server-window' 的值（server.el 调用时只传 buffer 一个参数）。"
  (let* ((frame (selected-frame))
         (candidates
          (cl-remove-if
           (lambda (w) (window-parameter w 'window-side))
           (window-list frame 'no-minibuf (frame-first-window frame))))
         (target (car (sort candidates
                            (lambda (a b)
                              (let ((ya (nth 1 (window-edges a nil t t)))
                                    (yb (nth 1 (window-edges b nil t t))))
                                (or (< ya yb)
                                    (and (= ya yb)
                                         (< (nth 0 (window-edges a nil t t))
                                            (nth 0 (window-edges b nil t t)))))))))))
    (if (window-live-p target)
        (progn
          (set-window-buffer target buffer)
          (select-window target)
          target)
      (display-buffer buffer '(display-buffer-pop-up-window)))))

(defun literal/display-or-focus (buffer action &optional select)
  "有则聚焦，无则按 ACTION 创建。
BUFFER 已在某窗口显示时，聚焦该窗口（SELECT 非 nil）；否则用 ACTION
（display-buffer 动作列表）创建新窗口。临时绑定 display-buffer-alist 为 nil，
避免递归匹配 display-buffer-alist 中的规则。"
  (when (buffer-live-p buffer)
    (if-let* ((win (get-buffer-window buffer)))
        (when select (select-window win))
      (let ((display-buffer-alist nil))
        (display-buffer buffer action)))))

*** GUI/TTY 混合 daemon 约束
:PROPERTIES:
:CUSTOM_ID: mixed-gui-tty-constraints
:END:

daemon 启动期 =(display-graphic-p)= 恒为 nil,所有依赖 display 类型的设置不能在启动时一次性决定,必须 per-frame。现代终端(foot/tmux)普遍支持真彩(24-bit),但 ef-themes 的主题背景会涂满整个 frame、遮挡终端的模糊背景,故本配置 TTY 下不加载 ef-themes,改用终端 ANSI 语义色(由 foot + darkman 配色方案统一驱动)覆盖前景,背景一律 =unspecified= 实现透明。

(defun literal/which-key--join-key (prefix key)
  "将 PREFIX 与 KEY 合并为 which-key 可用的键序列字符串。"
  (if (and prefix (> (length prefix) 0))
      (concat prefix " " key)
    key))

(defun literal/which-key--flatten-spec (prefix spec)
  "将 PREFIX 下的单个 SPEC 展开为 ((key . desc) ...) 列表。"
  (pcase spec
    (`(,key ,desc . ,children)
     (let ((full-key (literal/which-key--join-key prefix key)))
       (append (list (cons full-key desc))
               (literal/which-key--flatten-specs full-key children))))
    (`(,key . ,desc)
     (list (cons (literal/which-key--join-key prefix key) desc)))
    (_
     (error "无法识别的 which-key 描述项: %S" spec))))

(defun literal/which-key--flatten-specs (prefix specs)
  "将 PREFIX 下的 SPECS 展开为平铺的 ((key . desc) ...) 列表。"
  (apply #'append
         (mapcar (lambda (spec)
                   (literal/which-key--flatten-spec prefix spec))
                 specs)))

(defconst literal:which-key-replacements
  (literal/which-key--flatten-specs nil literal:which-key-description-spec)
  "Which-key 全局中文描述映射表。")

(defvar literal:which-key-descriptions-applied nil
  "避免重复写入 `which-key-replacement-alist'。")

(defun literal/which-key-apply-descriptions ()
  "将中文描述注册到 `which-key'。"
  (when (and (featurep 'which-key)
             (not literal:which-key-descriptions-applied))
    (dolist (entry literal:which-key-regexp-replacements)
      (setq which-key-replacement-alist
            (append which-key-replacement-alist (list entry))))
    (dolist (entry literal:which-key-replacements)
      (which-key-add-key-based-replacements (car entry) (cdr entry)))
    (dolist (mode-entry literal:which-key-major-mode-description-spec)
      (let ((mode (car mode-entry))
            (specs (cdr mode-entry)))
        (dolist (entry (literal/which-key--flatten-specs nil specs))
          (which-key-add-major-mode-key-based-replacements
            mode (car entry) (cdr entry)))))
    (setq literal:which-key-descriptions-applied t)))

(with-eval-after-load 'which-key
  ;; 先注册 literal/set-key 收集的延迟描述
  (literal/register-pending-wk-descs)
  ;; 再注册集中维护的描述
  (literal/which-key-apply-descriptions))

*** Frame 初始化

主题初始化时机按运行模式分流:

#+attr_org: :width 50
| 运行模式   | 触发时机            | 原因                            |
|------------+---------------------+---------------------------------|
| daemon     | 首个图形 frame 创建 | 启动期无 GUI frame,无法加载主题 |

(defun literal/graphic-frame-exists-p ()
  "返回当前会话中是否存在任意 GUI frame。"
  (catch 'found
    (dolist (frame (frame-list))
      (when (display-graphic-p frame)
        (throw 'found t)))
    nil))

(defun literal/display-supports-themes-p (&optional frame)
  "FRAME（默认 selected-frame）是否应加载 ef-themes 主题。
仅 GUI frame 加载主题；TTY 即使支持真彩也不加载——主题背景会遮挡终端的
模糊背景，TTY 改由 =literal/apply-terminal-faces= 用终端 ANSI 语义色覆盖。"
  (with-selected-frame (or frame (selected-frame))
    (display-graphic-p)))

(defun literal/color-scheme-init ()
  "初始化颜色方案适配：
1. 先从状态文件读取模式，立即应用主题
2. 异步验证 D-Bus，不一致则更新
3. 注册 D-Bus 监听器"
  (let ((initial-mode (or (literal/color-scheme-read-state)
                          (literal/color-scheme-get-from-dbus)
                          'dark)))
    (literal/color-scheme-apply-theme initial-mode)
    (setq literal/color-scheme--initialized t)
    (literal/color-scheme-register-listener)
    (run-with-idle-timer 1 nil
                         (lambda ()
                           (let ((dbus-mode (literal/color-scheme-get-from-dbus)))
                             (when (and dbus-mode
                                        (not (eq dbus-mode literal/color-scheme-current-mode)))
                               (message "[color-scheme] 状态文件与 D-Bus 不一致，更新为 %s" dbus-mode)
                               (literal/color-scheme-apply-theme dbus-mode)))))))

(defun literal/color-scheme-apply-for-frame (&optional frame)
  "在 FRAME 可用时补齐与主题相关的 frame 初始化。
主题是全局状态，不能在每个新 frame 上重复 `load-theme`，否则多 client
几乎同时创建 frame 时容易在 PGTK/GTK 初始化路径中崩溃。"
  (with-selected-frame (or frame (selected-frame))
    (when (literal/display-supports-themes-p)
      (when (fboundp 'ef-themes-take-over-modus-themes-mode)
        (ef-themes-take-over-modus-themes-mode 1))
      (if (not literal/color-scheme--initialized)
          (literal/color-scheme-init)
        (let ((target-theme (pcase literal/color-scheme-current-mode
                              ('dark literal/color-scheme-dark-theme)
                              ('light literal/color-scheme-light-theme)
                              (_ nil))))
          (when (and target-theme
                     (not (eq (car custom-enabled-themes) target-theme)))
            (ef-themes-load-theme target-theme)
            (message "[color-scheme] 新 frame 同步主题: %s" target-theme)))))))

  "延迟初始化颜色方案（daemon 模式首帧创建时触发）。

             (literal/display-supports-themes-p frame)
             (not literal/color-scheme--initialized))
    (with-selected-frame frame
      (require 'ef-themes)
      (literal/color-scheme-init)))
  nil)

    (progn
      (literal/add-frame-created-hook #'literal/color-scheme-delayed-init)
      (literal/add-frame-created-hook #'literal/color-scheme-apply-for-frame))

                    (with-selected-window window
                      (funcall refresher))

                            major-mode (error-message-string err))))))))))))

*** Apheleia 与 Flymake

Apheleia 用 =:hook (prog-mode . apheleia-mode)= 而非 =:defer t= 触发加载:后者没有可靠的首次加载入口,=apheleia-global-mode= 不会启用,保存时格式化在用户首次手动调 Apheleia 命令前都失效。进任何 prog buffer 就启用 global mode,保存时格式化立即生效。

Flymake 不需要 global 化——Eglot 在它管理的 buffer 内自动启用 =flymake-mode-hook=;非 LSP buffer 由各 major mode 的 =syntax-table= 提供诊断。全局跳转键 =M-g n/p= 与 =C-c l e= 在 [[#keybindings][前缀键体系]] 域统一绑定,不在此处 =:bind=,避免 flymake 未加载时键位悬空。

(defun literal/switch-project ()
  "切换到已知项目,并以 Dired 打开项目根目录。

(defun literal/open-project-folder ()
  "打开任意文件夹:切换 `default-directory' 并显示 Dired。

(defun literal/knowledge-archive-inbox-entry (category)
  "将 inbox.org 中光标所在的条目归档到 experiences/<CATEGORY>/ 目录。
slug 算法、PROPERTIES 模板与 reindex 触发全部由 agenote CLI 的
`inbox-archive' 子命令负责,Emacs 只负责收集 heading + body 并调 CLI。"
  (interactive
   (list (completing-read "归档分类: "
                          (or (condition-case nil
                                  (split-string
                                   (literal/agenote-call-string 'human "fields" "--category")
                                   "\n" t "[ \t]+")
                                (error nil))
                              '("emacs" "guix" "general" "gamedev" "devops" "research" "tooling"))
                          nil t)))
  (let* ((heading (or (org-get-heading t t t t) ""))
         (full-text (progn (org-copy-subtree) (car kill-ring)))
         (body (if (and full-text (string-match "^\\*+ [^\n]+\n:PROPERTIES:\n\\(?:[^\n]*\n\\)*?:END:\n"
                                                full-text))
                   (substring full-text (match-end 0))
                 (or full-text "")))
         (stdin (json-encode `[(("heading" . ,heading) ("body" . ,body))]))
         (result (literal/agenote-call 'human "inbox-archive"
                                       "--category" category "--stdin" "--prune")))
    (if (eq (plist-get result :status) 0)
        (progn
          (setq kill-ring (cdr kill-ring))
          (when (buffer-live-p (get-buffer literal:org-inbox-file))
            (with-current-buffer (get-buffer literal:org-inbox-file)
              (revert-buffer t t t)))
          (message "已归档: %s" (string-trim (plist-get result :stdout))))
      (message "归档失败: %s" (plist-get result :stderr)))))

#+end_src

inbox 归档的 body 提取逻辑: =org-copy-subtree= 把当前 subtree(含 heading 与 =PROPERTIES= 块)复制到 kill-ring,这里只取正文——从第一行 heading 之后到第一个 =:END:= 之间的 =PROPERTIES= 块被正则剥离,剩余作为 body 通过 =--stdin= 传给 CLI。CLI 完成归档后会 reindex + prune inbox,Emacs 端只需清理 kill-ring 并 =revert-buffer= 刷新 inbox buffer。

(defun literal/code--current-line-text ()
  "返回当前行文本,包含行尾换行。"
  (buffer-substring (line-beginning-position)
                    (line-beginning-position 2)))

(defun literal/code-duplicate-line ()
  "复制当前行并在下方粘贴。"
  (interactive)
  (let ((column (current-column))
        (line (literal/code--current-line-text))
        (insert-at (line-beginning-position 2)))
    (goto-char insert-at)
    (insert line)
    (goto-char insert-at)
    (move-to-column column)))

(defun literal/code-move-line-up ()
  "将当前行上移一行。"
  (interactive)
  (transpose-lines 1)
  (forward-line -2))

(defun literal/code-move-line-down ()
  "将当前行下移一行。"
  (interactive)
  (forward-line 1)
  (transpose-lines 1)
  (forward-line -1))

(defun literal/code-goto-definition ()
  "跳转到定义。"
  (interactive)
  (call-interactively #'xref-find-definitions))

(defun literal/code-goto-references ()
  "查找所有引用。"
  (interactive)
  (call-interactively #'xref-find-references))

(defun literal/code-jump-back ()
  "跳回之前的位置。"
  (interactive)
  (if (fboundp 'xref-go-back) (xref-go-back) (pop-mark)))

(defun literal/code-jump-forward ()
  "跳到下一个位置。"
  (interactive)
  (if (fboundp 'xref-go-forward) (xref-go-forward)
    (message "前进功能不可用")))

(defun literal/code-format-dwim ()
  "智能格式化:有选区格式化选区,否则格式化整个缓冲区。"
  (interactive)
  (if (use-region-p)
      (indent-region (region-beginning) (region-end))
    (cond
     ((and (fboundp 'eglot-managed-p) (eglot-managed-p))
      (eglot-format-buffer))
     (t
      (indent-region (point-min) (point-max))))))

(defun literal/code-comment-block ()
  "使用块注释包围选区。"
  (interactive)
  (if (use-region-p)
      (let ((comment-style 'multi-line))
        (comment-region (region-beginning) (region-end)))
    (message "请先选择要块注释的区域")))

*** In-region 补全

Corfu 提供轻量级的代码补全界面，接管 in-region 补全（替代 *Completions* 回退）; 延迟 0.2 秒加载以加速启动。

配置说明：
+ ~corfu-auto~ : 自动弹出补全
+ ~corfu-auto-prefix~ : 输入几个字符后触发补全
+ ~corfu-quit-no-match~ : 无匹配时的行为
+ ~corfu-cycle~ : 循环选择候选项

Cape 提供 ~Completion-At-Point~ 扩展后端（CAPF），与 Corfu 的 in-region 补全完全兼容。
各 capf 通过 buffer-local 追加到 =completion-at-point-functions= ，排在已有 major-mode / LSP capf 之后兜底。

可用 capf 清单（cape 2.7）：
+ ~cape-dabbrev~ : 当前 buffer 内动态缩写展开（覆盖旧 cape-symbol 功能）
+ ~cape-file~ :    字符串内路径补全
+ ~cape-history~ : 当前 minibuffer/region 历史回填
+ ~cape-keyword~ : 编程语言关键字补全（仅 prog-mode 有意义）
+ ~cape-line~ :    当前 buffer 内整行近似补全（性能略重，单独控制）
+ ~cape-tex~ :     LaTeX 数学符号补全（按需）

按模式分组配置 capf，避免 LSP 管理的 prog-mode 被无关 capf 干扰。

~Consult-flymake~ - Flymake 错误浏览(Emacs 29+ 内置,无独立包)
consult 内置 =consult-flymake= 命令,通过 Consult 界面浏览和跳转 Flymake 诊断。
M-s f 在前缀键体系绑定到 =consult-flymake= 。

(defconst literal/completion:ispell-word-list-candidates
  (delq nil
        (list (and (getenv "GUIX_PROFILE")
                   (expand-file-name "share/dict/words" (getenv "GUIX_PROFILE")))
              (expand-file-name "~/.guix-home/profile/share/dict/words")
              (expand-file-name "~/.guix-profile/share/dict/words")
              "/run/current-system/profile/share/dict/words"
              "/usr/share/dict/words"
              "/usr/share/dict/web2"))
  "候选 plain word list 路径。`miscfiles' 通常提供 share/dict/words。")

(defvar literal/completion--ispell-word-list nil
  "已发现的 plain word list 文件。nil 表示尚未找到。")

(defun literal/completion-ispell-find-word-list ()
  "返回可用于 `ispell-alternate-dictionary' 的 plain word list 文件。"
  (or literal/completion--ispell-word-list
      (setq literal/completion--ispell-word-list
            (seq-find #'file-readable-p literal/completion:ispell-word-list-candidates))))

(defun literal/completion-setup-ispell-program ()
  "配置可用的 ispell 后端命令。
两者均不可用时不修改 ispell 配置。"
  (when-let* ((program (or literal:executable-aspell
                           literal:executable-hunspell)))
    (setq ispell-program-name program)
    (when literal:executable-aspell
      (setq ispell-extra-args '("--sug-mode=ultra")))))

        (setq-local ispell-alternate-dictionary word-list)

(defun literal/completion-setup-cape-org ()
  "Org mode 的 cape capf 组合：只保留 file（路径补全）。
由于 org-mode 是 text-mode 的衍生模式，`text-mode-hook' 会先运行并插入
cape-dabbrev / cape-line；因此本函数先移除这两个 capf，再追加 cape-file，

补全仍保留。

  (setq-local completion-at-point-functions
              (remove #'cape-line
                      (remove 'cape-line
                              (remove #'cape-dabbrev
                                      (remove 'cape-dabbrev
                                              completion-at-point-functions)))))
  (literal/completion-cape-add-capfs '(cape-file))
  (when (boundp 'corfu-auto)
    (setq-local corfu-auto nil)))

** Dashboard
:PROPERTIES:
:CUSTOM_ID: dashboard
:END:

启动仪表盘:集成待办列表、文件管理器、前缀键概览和鼠标工作流提示。

Dashboard 不依赖第三方 =dashboard= 包:自建 =literal/dashboard-mode= (derived from =special-mode=) + 纯函数渲染引擎,全部基于列模型(=string-width=),GUI 与 TTY / batch 单位一致,所有宽度算术可证明。Dashboard 依赖知识库、帮助、颜色方案和 frame 接口,因此固定在最后一个配置域加载。跨域调用保留 =fboundp= 守卫,但正常路径由文档顺序保证依赖已就绪。

*** 六层架构

#+attr_org: :width 70
| 层   | noweb ref                   | 职责                                       | 副作用       |
|------+-----------------------------+--------------------------------------------+--------------|
| 核心 | =system/dashboard-core=       | face 定义、buffer 常量、TTL 缓存原语       | face         |
| 测量 | =system/dashboard-measure=    | 列模型:measure / pad / truncate            | 纯函数       |
| 注册 | =system/dashboard-registry= | 布局声明 + 卡片 spec 注册表 + tier 判定    | 纯函数       |
| 引擎 | =system/dashboard-engine=     | 三态卡片 + 错误隔离 + 响应式拼接           | 纯函数       |
| 卡片 | =system/dashboard-cards=      | 7 张卡片 fetch/render + IO/计算            | IO           |
| 生命 | =system/dashboard-lifecycle=  | mode、render、frame 状态机、防抖、主题刷新 | 唯一副作用层 |

设计要点:

- 卡片契约: =fetch= 无参 → =(:ok DATA)|(:error MSG)|loading= 三态; =render (DATA WIDTH)= 是纯函数返回行列表,行宽 ≤ WIDTH。空 / 加载中 / 错误 三态可区分,杜绝"假空状态"。
- 错误隔离: 单张卡片任何异常被引擎包成错误卡片,绝不拖垮整页。
- 测量层: 全部基于 =string-width= 的列模型,GUI 与 TTY / batch 统一,所有宽度算术单位一致(见 [[测量层][测量层章节]]为何放弃像素测量)。
- 零临时 buffer: 卡片渲染器是纯函数,拼接是字符串操作,消除了旧实现每刷新 6 次 =generate-new-buffer= 的开销,且每张卡片可独立进 ERT。
- 宽度显式传参: 删除旧的 =--forced-width= / =--override-margin= 动态绑定,宽度沿调用链显式传递。

*** 测量层

测量层是右列对齐的基石,全部基于 =string-width= 的列模型,GUI 与 TTY / batch 统一。这放弃了原设计的 GUI 像素测量方案——像素值与卡片宽度算术(=window-body-width= 列数)、规则线(=make-string= 按字符数)的单位不一致曾导致标题无间距、规则线截断、banner 居中失效。列模型下 Nerd Font PUA 图标至多 1 列误差(见重设计文档 §1 #9),但所有算术单位一致、布局不串位。

#+begin_src emacs-lisp :noweb-ref system/dashboard-measure :tangle no

(defun literal/dashboard--measure (text)
  "返回 TEXT 的显示宽度（列数）。
全部基于 `string-width'，GUI 与 TTY / batch 统一为列模型。

为何不用像素测量（`string-pixel-width'）：原设计曾尝试 GUI 像素 + TTY 列双模，
但 `string-pixel-width' 返回像素值，而卡片宽度算术（=window-body-width= 列数
减边距）、规则线（`make-string' 按字符数）等全部以列为单位。两种单位混用导致
标题无间距、规则线截断、banner 居中失效（见 commit 修复前的 GUI 截图）。列模型
下 `string-width' 对 Nerd Font PUA 图标算 1 列但字形可能占 2 列像素，存在至多
1 列的右栏平移误差（见重设计文档 §1 #9），但所有算术单位一致、布局不会串位。
该 1 列误差留作后续 per-frame buffer 增强的附带改进点。"
  (string-width (or text "")))

(defun literal/dashboard--fill (width)
  "生成 WIDTH 列宽的 ASCII 空格填充。"
  (if (<= width 0)
      ""
    (make-string width ?\s)))

(defun literal/dashboard--pad-to (text width)
  "把 TEXT 右填充到恰好 WIDTH 列（不足补空格，超出原样返回不截断）。"
  (let ((slack (- width (literal/dashboard--measure text))))
    (if (> slack 0)
        (concat text (literal/dashboard--fill slack))
      text)))

(defun literal/dashboard--truncate (text width)
  "截断 TEXT 到 WIDTH 列，超宽补 \"...\"。
三个 ASCII 点宽度恒定（3 列），全局统一，避免 \"…\" 的宽度随字体漂移。"
  (if (<= (literal/dashboard--measure text) width)
      text
    (truncate-string-to-width text width 0 nil "...")))

(defun literal/dashboard--spread (left right width)
  "LEFT 靠左、RIGHT 靠右，中间填充到恰好 WIDTH 列。"
  (let ((gap (- width
                (literal/dashboard--measure left)
                (literal/dashboard--measure right))))
    (concat left (literal/dashboard--fill (max 0 gap)) right)))

#+end_src

*** 渲染引擎

引擎统一包边框（左竖线 + 标题 + 规则线）、三态降级与错误隔离,卡片渲染器无需重复处理这些。单张卡片任何异常被隔离为错误卡片(见 A2 修复),绝不拖垮整页。

#+begin_src emacs-lisp :noweb-ref system/dashboard-engine :tangle no
(defun literal/dashboard--bar ()
  "返回左竖线字符（带边框 face）。宽度恒为 1 列。"
  (propertize "|" 'face 'literal/dashboard-card-bar))

(defun literal/dashboard--rule (body-width)
  "返回标题下分隔线（body-width 列的 `-'，带边框 face）。
列模型下 `-' 的 char-width 恒为 1，make-string N 个 `-' 的视觉宽 == N 列，
恰好等于内容区；像素模式下 `-' 同样可控。规则线右端无需对齐（边框设计本就
无右边缘），所以按列铺满已足够。"
  (propertize (make-string (max 0 body-width) ?-)
              'face 'literal/dashboard-card-bar))

(defun literal/dashboard--card-lines (spec width)
  "把 SPEC 渲染成 WIDTH 宽的完整卡片行列表（含边框/标题/规则线/正文）。
单个卡片的任何异常（fetch 抛错、render 抛错）都被隔离为错误卡片，
不影响同排其他卡片。返回行字符串列表，行间不含空行（空行由布局层加）。"
  (let* ((body-width (max 0 (- width 2)))
         (bar (literal/dashboard--bar))
         (state (condition-case err
                    (funcall (plist-get spec :fetch))
                  (error `(:error ,(error-message-string err)))))
         (body
          (pcase state
            (`(:ok ,data)
             (condition-case err
                 (or (funcall (plist-get spec :render) data body-width)
                     (list ""))
               (error
                (list (propertize (format "渲染失败：%s"
                                          (error-message-string err))
                                  'face 'literal/dashboard-error)))))
            (`(:error ,msg)
             (list (propertize (format "󰙦 暂不可用：%s" msg)
                               'face 'literal/dashboard-error)
                   (propertize "按 g 重试"
                               'face 'literal/dashboard-dim)))
            ('loading
             (list (propertize "... 加载中"
                               'face 'literal/dashboard-dim)))
            (_ (list ""))))
         (icon (propertize (or (plist-get spec :icon) "")
                           'face (or (plist-get spec :accent)
                                     'literal/dashboard-card-title)))
         (title-text (propertize (or (plist-get spec :title) "")
                                 'face 'literal/dashboard-card-title))
         (key-text (propertize (or (plist-get spec :key) "")
                               'face 'literal/dashboard-dim))
         (title-line (literal/dashboard--spread
                      (if (string-empty-p icon)
                          title-text
                        (concat icon " " title-text))
                      key-text
                      body-width))
         (rule (literal/dashboard--rule body-width)))
    (append
     (list bar)
     (list (concat bar " " title-line))
     (list (concat bar rule))
     (mapcar (lambda (line)
               (let ((trimmed (literal/dashboard--truncate line body-width)))
                 (concat bar " " trimmed)))
             body)
     (list bar))))

(defun literal/dashboard--render-row (card-ids total-width)
  "渲染一行（1 或 2 张卡片），返回行字符串列表。
2 张卡片时等高拼接：短卡片补带左竖线的空延续行；右列不补尾部填充，
消除行尾空白。"
  (pcase (length card-ids)
    (1 (literal/dashboard--card-lines
        (literal/dashboard--card-spec (car card-ids)) total-width))
    (2 (let* ((left-spec (literal/dashboard--card-spec (nth 0 card-ids)))

(defun literal/dashboard--compact-path (path)
  "将 PATH 中间目录压缩为首字母，首尾保留完整。
隐藏目录（以 . 开头）取其第二个字符，避免 ~/./e 这种怪异的路径。"
  (let* ((parts (split-string path "/" t))
         (len (length parts)))
    (if (<= len 3)
        path
      (let* ((head (car parts))
             (file (car (last parts)))
             (middle (cl-subseq parts 1 (1- len)))
             (compressed (mapcar (lambda (d)
                                   (cond ((string-empty-p d) d)
                                         ((string-prefix-p "." d)
                                          (if (> (length d) 1)
                                              (concat "." (substring d 1 2))
                                            d))
                                         (t (substring d 0 1))))
                                 middle)))
        (concat head "/"
                (mapconcat #'identity compressed "/")
                "/" file)))))

(defun literal/dashboard--compute-todo-items (max-items)
  "实际扫描 `org-agenda-files' 计算未完成 TODO 项（无缓存）。
每个条目记录 (STATE TITLE DATE-INFO FILE POS)，POS 为 heading 在 file 中的
字符位置，用于点击跳转。

`ask-user-about-supersession-threat' 是函数，用 `cl-letf' 绑定其
`symbol-function'（而非 let 绑定变量——那是 dead binding，无法抑制磁盘变更
询问）。fresh daemon 首次渲染的同步 `require' 由 fetch 层的 loading 态兜底。"
  (require 'org-agenda)
  (require 'org)
  (let ((today-abs (calendar-absolute-from-gregorian (calendar-current-date)))
        (count 0)
        raw-items)
    (when (and (fboundp 'org-agenda-files)
               (ignore-errors (org-agenda-files)))
      (dolist (file (org-agenda-files))
        (when (and (< count (* max-items 2)) (file-exists-p file))
          (let ((buf (cl-letf (((symbol-function 'ask-user-about-supersession-threat)
                                (lambda (&rest _) nil)))
                       (find-file-noselect file))))
            (unwind-protect
                (with-current-buffer buf
                  (org-with-wide-buffer
                   (goto-char (point-min))
                   (while (and (< count (* max-items 2))
                               (re-search-forward org-heading-regexp nil t))
                     (let* ((heading-pos (match-beginning 0))
                            (components (org-heading-components))
                            (todo-state (nth 2 components))
                            (title (nth 4 components))
                            (scheduled (org-entry-get (point) "SCHEDULED"))
                            (deadline (org-entry-get (point) "DEADLINE")))
                       (when (and todo-state
                                  (not (member todo-state '("DONE" "CANCELLED"))))
                         (let* ((deadline-abs (and deadline
                                                   (org-time-string-to-absolute deadline)))
                                (scheduled-abs (and scheduled
                                                    (org-time-string-to-absolute scheduled)))
                                (urgency
                                 (cond
                                  ((and deadline-abs (< deadline-abs today-abs)) 0)
                                  ((and deadline-abs (= deadline-abs today-abs)) 1)
                                  ((and scheduled-abs (< scheduled-abs today-abs)) 2)
                                  (deadline-abs 3)
                                  (scheduled-abs 4)
                                  (t 5))))
                           (push (list urgency todo-state title deadline
                                       scheduled deadline-abs scheduled-abs
                                       file heading-pos)
                                 raw-items)
                           (setq count (1+ count)))))))))
            (when (buffer-live-p buf)
              (kill-buffer buf))))))
    (setq raw-items
          (sort raw-items
                (lambda (a b)
                  (or (< (car a) (car b))
                      (and (= (car a) (car b))
                           (and (nth 5 a) (nth 5 b)
                                (< (nth 5 a) (nth 5 b))))))))
    (let ((result nil)
          (n 0))
      (dolist (item raw-items)
        (when (< n max-items)
          (let* ((todo-state (nth 1 item))
                 (title (nth 2 item))
                 (deadline-str (nth 3 item))
                 (scheduled-str (nth 4 item))
                 (deadline-abs (nth 5 item))
                 (scheduled-abs (nth 6 item))
                 (file (nth 7 item))
                 (pos (nth 8 item))
                 (state-face (literal/dashboard--todo-state-face todo-state))
                 (date-info (literal/dashboard--todo-date-info
                             today-abs deadline-str scheduled-str
                             deadline-abs scheduled-abs)))
            (push (list (propertize (concat todo-state "  ") 'face state-face)
                        (propertize title 'face 'literal/dashboard-todo-title)
                        date-info
                        file pos)
                  result)
            (setq n (1+ n)))))
      (nreverse result))))

         (row-lines nil))
    (dotimes (i rows-count)
      (let* ((left-idx i)
             (right-idx (+ i rows-count))
             (left (nth left-idx prefix-bindings))
             (right (when (< right-idx total) (nth right-idx prefix-bindings)))
             (left-col (literal/dashboard--format-shortcut-column left col-width))
             (right-col (literal/dashboard--format-shortcut-column right col-width)))
        (push (concat prefix
                      left-col
                      (literal/dashboard--fill col-gap)
                      right-col)
              row-lines)))
    (nreverse row-lines)))

(defun literal/dashboard--status-line ()
  "返回 banner 下方的状态行字符串，或 nil。
汇总 Emacs 版本、启动耗时与当前主题模式；任何字段缺失该槽显示 ?。
全部 nil 守卫，batch / fresh daemon 下安全。

`(emacs-version)' 返回 \"GNU Emacs 31.0.90 ...\"，正则需匹配 GNU 前缀。
启动耗时用秒数自行格式化为紧凑形式（`(emacs-init-time)' 在 batch 下返回
原始秒数字符串如 \"0.002s\"，交互下返回 \"0.5 seconds\"，统一格式化避免漂移）。"
  (let* ((emacs-ver (and (fboundp 'emacs-version)
                         (string-match "GNU Emacs \\([0-9.]+\\)" (emacs-version))
                         (match-string 1 (emacs-version))))
         (init-seconds (and (fboundp 'emacs-init-time)
                            (condition-case nil (emacs-init-time) nil)))
         (init-time (cond
                     ((null init-seconds) "?")
                     ;; 数字开头（batch 返回 "0.002s" / 交互返回 "0.5 seconds"）
                     ((string-match "\\`\\([0-9.]+\\)" init-seconds)
                      (let ((secs (string-to-number (match-string 1 init-seconds))))
                        (if (>= secs 1)
                            (format "%.1fs" secs)
                          (format "%.0fms" (* secs 1000)))))
                     (t init-seconds)))
         (mode (when (boundp 'literal/color-scheme-current-mode)
                 (symbol-name literal/color-scheme-current-mode))))
    (format "Emacs %s · 启动 %s · %s"
            (or emacs-ver "?")
            init-time
            (or mode "?"))))

(defun literal/dashboard--banner-min-width ()
  "banner ASCII art 居中显示所需的最小窗口列宽。
随显示类型切换度量：GUI 用字符数（box-drawing 占 1 字符单元），TTY 用
string-width（占 2 列），与 `literal/dashboard--insert-banner' 一致。
另留 10 列左右边距，避免 art 紧贴窗口边缘。"
  (let* ((art-max-line "███████╗███╗   ███╗ █████╗  ██████╗███████╗")
         (art-w (if (display-graphic-p)
                    (length art-max-line)
                  (string-width art-max-line))))
    (+ art-w 10)))

(defun literal/dashboard--insert-banner (width)
  "在当前 buffer 插入居中的 banner（art + 状态行）。
art 各行左边缘对齐到最长行的居中 padding（figlet art 要求行间左对齐），
状态行单独按自身宽度居中到窗口中央。窄窗口（低于
`literal:dashboard-banner-min-width'）降级为纯文本单行。

+ art 居中的宽度度量随显示类型切换: GUI 等宽字体(如 DejaVu Sans Mono)下
  box-drawing 字符(█ ╗ ╝ ═)通常占 1 个字符单元,故用 =length= 算 pad;
  TTY / batch 下这些字符占 2 列终端单元, =string-width= 报告的 2 更准确,
  故用 =string-width=。用错误的度量会导致 banner 视觉偏左(GUI 用 string-width)
  或偏右(TTY 用 length)。状态行是混合文本(ASCII + CJK + ·),两种环境都用
  =string-width=。

空行（art 首尾的 \"\"）插入纯换行，不插入前导 padding 空格，避免产生
满是空格的视觉空行。"
  (let ((banner-face 'font-lock-keyword-face)
        (status-face 'literal/dashboard-dim)
        (gui-p (display-graphic-p)))
    (if (< width (literal/dashboard--banner-min-width))
        (let ((pad (max 0 (/ (- width (length "LITERAL")) 2))))
          (insert "\n"
                  (literal/dashboard--fill pad)
                  (propertize "LITERAL" 'face banner-face)
                  "\n\n"))
      (let* ((lines (literal/dashboard--ascii-banner-lines))
             (status (literal/dashboard--status-line))
             (art-lines (cl-remove-if
                         (lambda (l) (or (string-empty-p l) (equal l status)))
                         lines))
             ;; GUI 用 length(box-drawing 占 1 字符单元),TTY 用 string-width(占 2)
             (art-width-fn (if gui-p #'length #'literal/dashboard--measure))
             (art-max-w (apply #'max (mapcar art-width-fn art-lines)))
             (art-pad (max 0 (/ (- width art-max-w) 2))))
        (insert "\n")
        (dolist (line lines)
          (cond
           ((string-empty-p line) (insert "\n"))
           ((equal line status)
            ;; 状态行用 string-width 居中（混合文本，两环境一致）
            (let ((status-pad (max 0 (/ (- width (literal/dashboard--measure line)) 2))))
              (insert (literal/dashboard--fill status-pad)
                      (propertize line 'face status-face)
                      "\n")))
           (t
            ;; art 行统一 pad
            (insert (literal/dashboard--fill art-pad)
                    (propertize line 'face banner-face)
                    "\n"))))
        (insert "\n")))))

(defun literal/dashboard-render ()
  "自有渲染入口：erase → banner → 布局引擎 → insert。
不经过第三方 dashboard 包任何函数。失败时保留旧 buffer 内容（两阶段：先在
当前 buffer 构造，任何 error 由上层 condition-case 兜底）。"
  (interactive)
  (let ((buf (get-buffer-create literal:dashboard-buffer-name)))
    (prog1 buf
      (with-current-buffer buf
        (unless (derived-mode-p 'literal/dashboard-mode)
          (literal/dashboard-mode))
        (let* ((window (or (get-buffer-window buf t)
                           (frame-selected-window)))
               (width (if window (window-body-width window) (frame-width)))
               (content-width (max 1 (- width (* 2 literal:dashboard-dual-margin))))
               (margin-str (literal/dashboard--fill literal:dashboard-dual-margin)))
          (let ((inhibit-read-only t))
            (erase-buffer)
            (literal/dashboard--insert-banner width)
            (dolist (line (literal/dashboard--build-lines content-width))
              (insert margin-str line "\n"))
            (goto-char (point-min))
            (setq-local literal/dashboard--dirty nil)))))))

(define-derived-mode literal/dashboard-mode special-mode "Dashboard"
  "自建 dashboard mode（derived from `special-mode'）。
不依赖第三方 dashboard 包：渲染完全由 `literal/dashboard-render' 驱动。"
  (setq-local truncate-lines t)
  (literal/dashboard--apply-fixed-pitch)
  (setq-local cursor-type nil
              line-spacing 0.15
              show-trailing-whitespace nil)
  (when (fboundp 'hl-line-mode) (hl-line-mode -1))
  (when (boundp 'display-line-numbers-mode) (display-line-numbers-mode -1))
  (buffer-disable-undo)
  (let ((map (make-sparse-keymap)))
    (define-key map "g" #'literal/dashboard-render)
    (define-key map "n" #'next-line)
    (define-key map "p" #'previous-line)
    (define-key map "q" #'quit-window)
    (use-local-map map)))

(defun literal/dashboard--placeholder-buffer-p (buffer)
  "判断 BUFFER 是否仍是 client frame 的占位缓冲区。"
  (when (buffer-live-p buffer)
    (with-current-buffer buffer
      (and (not (minibufferp buffer))
           (not buffer-file-name)
           (or (member (buffer-name buffer)
                       literal:dashboard-placeholder-buffer-names)
               (and (eq major-mode 'fundamental-mode)
                    (string-prefix-p " " (buffer-name buffer))))))))

  "判断 BUFFER 是否正在服务 emacsclient / with-editor 编辑请求。"
  (when (buffer-live-p buffer)
    (with-current-buffer buffer
      (or (bound-and-true-p server-buffer-clients)
          (bound-and-true-p with-editor-mode)
          (derived-mode-p 'git-commit-mode)))))

(defun literal/dashboard--single-window-frame-p (frame)
  "判断 FRAME 是否仍只有一个普通窗口。
Dashboard 只接管这种新 client frame 的空白启动状态；若 Magit、提交编辑器
或其他命令已经分裂窗口，则保留现有窗口配置。"
  (and (frame-live-p frame)
       (= (length (window-list frame 'no-minibuf)) 1)))

(defun literal/dashboard--refresh-if-visible ()
  "若 dashboard buffer 在某可见窗口，则立即重渲染。"
  (let* ((buf (get-buffer literal:dashboard-buffer-name))
         (window (and buf (get-buffer-window buf t))))
    (when (and window (window-live-p window))
      (with-selected-window window
        (literal/dashboard-render)))))

(defvar literal/dashboard--refresh-timer nil
  "dashboard 可见性刷新的 pending idle timer。")

(defun literal/dashboard--refresh-visible-now ()
  "立即检查所有可见 dashboard 窗口，dirty 或宽度变化者重新渲染。"
  (when (timerp literal/dashboard--refresh-timer)
    (cancel-timer literal/dashboard--refresh-timer))
  (setq literal/dashboard--refresh-timer nil)
  (dolist (window (window-list nil 'no-minibuf))
    (let ((buffer (window-buffer window)))
      (when (buffer-live-p buffer)
        (with-current-buffer buffer
          (when (and (derived-mode-p 'literal/dashboard-mode)
                     (or literal/dashboard--dirty t))
            (with-selected-window window
              (literal/dashboard-render))))))))

(defun literal/dashboard--maybe-refresh-visible (_frame)
  "若可见 dashboard 标记 dirty，则延迟重新渲染（合并连续尺寸变化）。"
  (when (timerp literal/dashboard--refresh-timer)
    (cancel-timer literal/dashboard--refresh-timer))
  (setq literal/dashboard--refresh-timer
        (run-with-idle-timer literal:dashboard-refresh-debounce
                             nil
                             #'literal/dashboard--refresh-visible-now)))

(defun literal/dashboard--display-in-frame (frame)
  "在 FRAME 中显示并刷新 dashboard。"
  (when (and (frame-live-p frame)
             (literal/dashboard--single-window-frame-p frame))
    (with-selected-frame frame
      (let ((buf (get-buffer-create literal:dashboard-buffer-name)))
        (when-let* ((window (frame-selected-window frame)))
          (with-current-buffer buf
            (unless (derived-mode-p 'literal/dashboard-mode)
              (literal/dashboard-mode)))
          (set-window-buffer window buf)
          (with-selected-window window
            (literal/dashboard-render)))))))

(defconst literal:dashboard-open-idle-delay 0.05
  "新建 client frame 后等待 dashboard 判定的 idle 秒数。")

(defconst literal:dashboard-open-retries 2
  "若首次判定过早，dashboard 允许追加重试的次数。")

(defun literal/dashboard--frame-startup-state (frame)
  "返回 FRAME 当前的启动缓冲区状态。"
  (cond
   ((not (literal/dashboard--single-window-frame-p frame)) 'busy)
   (t
    (if-let* ((window (and (frame-live-p frame)
                           (frame-selected-window frame)))
              (buffer (window-buffer window)))
        (with-current-buffer buffer
          (cond
           ((derived-mode-p 'literal/dashboard-mode) 'dashboard)
           ((literal/dashboard--server-edit-buffer-p buffer) 'server-edit)
           ((buffer-file-name) 'file)
           ((literal/dashboard--placeholder-buffer-p buffer) 'placeholder)
           (t 'busy)))
      'pending))))

(defun literal/dashboard--run-open-check (frame retries-left)
  "根据 FRAME 当前状态决定是否显示或重试 dashboard。"
  (when (frame-live-p frame)
    (set-frame-parameter frame 'literal-dashboard-open-scheduled nil)
    (pcase (literal/dashboard--frame-startup-state frame)
      ('placeholder
       (literal/dashboard--display-in-frame frame))
      ('dashboard
       (when-let* ((window (frame-selected-window frame)))
         (with-selected-window window
           (literal/dashboard-render))))
      ('file nil)
      ('server-edit nil)
      ('busy nil)
      (_
       (when (> retries-left 0)
         (literal/dashboard--schedule-open-for-frame
          frame (1- retries-left)))))))

(defun literal/dashboard--schedule-open-for-frame (frame retries-left)
  "为 FRAME 安排一次 dashboard 打开检查。RETRIES-LEFT 为剩余重试次数。"
  (when (and (frame-live-p frame)
             (not (frame-parameter frame 'literal-dashboard-open-scheduled)))
    (set-frame-parameter frame 'literal-dashboard-open-scheduled t)
    (run-with-idle-timer
     literal:dashboard-open-idle-delay nil
     (lambda (idle-frame idle-retries-left)
       (when (frame-live-p idle-frame)
         (literal/dashboard--run-open-check idle-frame idle-retries-left)))
     frame retries-left)))

仅当该 frame 仍停留在占位缓冲区时接管，避免覆盖 `emacsclient FILE`。"
  (literal/dashboard--schedule-open-for-frame
   (or frame (selected-frame))
   literal:dashboard-open-retries))

**** per-frame buffer（修 A5 多 frame 宽度抖动）

当前所有 =literal/dashboard-mode= 窗口共享同一 =*dashboard*= buffer,见本节引言的[多 frame 限制]。两个不同宽度的 client frame 互相触发重渲时内容宽度以最后触发者为准,偶发抖动。

目标:每个 frame 持有独立的 dashboard buffer(名如 =*dashboard*<N>=,N 为 frame 内部 id),互不影响。

实现要点:

1. =(literal/dashboard--frame-buffer frame)=:返回 frame 关联的 buffer,不存在则按命名约定创建。
2. frame parameter =literal-dashboard-buffer= 持有 buffer 引用;=delete-frame-functions= 清理对应 buffer(避免泄漏)。
3. =--refresh-if-visible= / =--display-in-frame= / =--maybe-refresh-visible= 全部从"操作唯一 =*dashboard*="改为"按 frame 取各自 buffer"。
4. =literal:dashboard-buffer-name= 从常量改为函数 =literal/dashboard--buffer-name-for-frame=。

风险:=C-x 5 2= 复制 frame 时新 frame 继承旧 buffer 引用,需显式重置 frame parameter。=winner-undo= / side-window 路由不涉及(buffer 在 frame 内的窗口只是显示层)。

**** knowledge 异步拉取

当前 =--fetch-knowledge= 同步调 =literal/knowledge-list-cards=(经 agenote CLI),fresh daemon 首次渲染时该 CLI 启动有可感知延迟。todo 已用 =--defer-until-org= + loading 态异步化,knowledge 可同样处理。

目标:缓存过期时发起 =make-process= 异步拉取,先返回 stale 或 =loading=,sentinel 解析 JSON → 写缓存 → =--refresh-if-visible=。

实现要点:

1. =(literal/dashboard--knowledge-start-async)=:幂等,进程已在跑则跳过。
2. sentinel 解析 JSON 成功后 =(setq literal/dashboard--knowledge-cache (cons ...))= 并置 =--dirty=。
3. fetch 三态:缓存有效 → =(:ok data)=;失效且有进程跑 → =loading= 或返回 stale;失效且无进程 → 发起 + =loading=。
4. =--org-defer-pending= 同款幂等标志防止并发拉取。

风险:agenote CLI JSON schema 变动需同步;进程异常退出 sentinel 要清标志位,否则永久 loading。参考 =literal/agenote-call-async=(定义于 [[#process-helper][共享进程辅助]],knowledge 域已多处使用)已验证的 async 模式。

**** 标题带计数

当前标题固定显示(如"待办"),fetch 返回的是已截断的 max-items 条,丢失了 total。文档 §4.6 称此项"成本为零",但实际需改 fetch 协议。

目标:标题显示 =󰃭 待办 (3)=,数字为实际未完成数(可能多于显示条数)。

实现要点(二选一):

- *方案 A(改协议)*:fetch 返回 =(:ok (DATA . TOTAL))=,render 接收 =data= 与 =total=。影响 7 张卡片 + 引擎 =--card-lines= 标题构造。
- *方案 B(spec 附 count 函数)*:卡片 spec 增加 =:count= 槽(无参 → 数字或 nil),引擎调它得 total 拼 title。改动更局部,仅引擎 + 需计数的卡片。

**** Ghostel pre-spawn 的 with-editor 适配

with-editor 3.5.2 的公开导出函数只支持已经存在的 shell/term/vterm 进程；Ghostel 的 pre-spawn hook 运行在进程创建前。当前必须复用宏内部 setup，删除条件是 Ghostel 或 with-editor 提供公开的 pre-spawn 环境构造接口。

#+begin_src emacs-lisp :noweb-ref compat/ghostel-with-editor :tangle no
(defun literal/ghostel-export-git-editor ()
  "在 Ghostel 创建进程前把 GIT_EDITOR 注入动态 `process-environment'。"
  (when (fboundp 'with-editor--setup)
    (let ((with-editor--envvar "GIT_EDITOR"))
      (with-editor--setup))))
#+end_src

**** Which-key 中文宽度与网格布局补丁

which-key 3.6.1 已内置 =string-width= 计算，但自动列数在底部 side-window 下仍可能导致中文描述列宽不等。补丁用 =advice-add :override= 替换 =which-key--pad-column=，在固定网格布局下按等宽分栏。加 =fboundp= 守卫：which-key 已并入 Emacs 30+ 内置，未来若内部函数改名/移除，静默跳过。

#+begin_src emacs-lisp :noweb-ref compat/which-key-width :tangle no
(defun literal/which-key--pad-column-fixed-width (col-keys available-width)
  "为 COL-KEYS 生成更适合中文的 which-key 列布局。
AVAILABLE-WIDTH 为当前 which-key 可用宽度。固定网格布局下按等宽分栏,
非网格布局下按 which-key 原生逻辑但用 `string-width' 计算。"
  (let* ((grid-layout-p (literal/which-key--fixed-grid-layout-p))
         (col-key-width (+ which-key-add-column-padding
                           (which-key--max-len col-keys 0)))
         (col-sep-width (which-key--max-len col-keys 1))
         (target-col-width (literal/which-key--target-grid-column-width available-width))
         (desc-space (max 1 (- target-col-width col-key-width col-sep-width)))
         (col-desc-width (if grid-layout-p
                             desc-space
                           (max which-key-min-column-description-width desc-space)))
         (col-width (+ col-key-width col-sep-width col-desc-width))
         (col-format (concat "%" (int-to-string col-key-width) "s%s%s")))
    (cons col-width
          (mapcar
           (pcase-lambda (`(,key ,sep ,desc ,_doc))
             (let* ((display-desc (truncate-string-to-width
                                   desc col-desc-width 0 nil which-key-ellipsis))
                    (padding (max 0 (- col-desc-width (string-width display-desc)))))
               (concat
                (format col-format key sep display-desc)
                (make-string padding ?\s))))
           col-keys))))

(defun literal/which-key-install-width-fix ()
  "安装 Which-key 中文宽度与网格布局补丁。
加 fboundp 守卫:which-key 已并入 Emacs 30+ 内置,未来若内部函数改名/移除,
静默跳过而非崩溃。"
  (when (fboundp 'which-key--pad-column)
    (advice-add 'which-key--pad-column :override
                #'literal/which-key--pad-column-fixed-width)))
#+end_src

**** arei .elc 与 buffer-local-set-state 不兼容(bug#80812)

arei 0.9.7 的 .elc(Guix 用旧版 Emacs byte-compile)与 Emacs 31 的 =buffer-local-set-state= 宏不兼容:arei-mode 第 356 行调用该宏,但旧 .elc 把它当函数调用,参数 =(sesman-system 'Arei)= 被整体传给 =buffer-local-set-state--get= → =wrong-type-argument (symbolp 'Arei)= 。此错误在 =scheme-mode-hook= 里抛出,会中断整个 scheme-mode 初始化,导致 font-lock 失效 → Scheme 代码不染色。

#+begin_src emacs-lisp :noweb-ref emacs31/arei-compat :tangle no
(defun literal--load-arei-from-source ()
  "从 .el 源强制加载 arei,绕过与 Emacs 31 不兼容的 .elc。
GNU store 只读无法删 .elc,且 .el/.elc 时间戳相同(均为 epoch 1),
`load-prefer-newer' 无效,故需显式从 .el 加载。返回 t 若成功。"
  (condition-case err
      (let* ((found (locate-library "arei"))
             (arei-el (when found
                        (concat (file-name-sans-extension found) ".el"))))
        (when (and arei-el (file-exists-p arei-el))
          (load arei-el nil t t)
          t))
    (error
     (message "[literal] arei 从源加载失败: %S" err)
     nil)))

(defun literal--maybe-enable-arei-mode ()
  "在真实 Scheme 编辑 buffer 中启用 `arei-mode'(跳过 org src 临时 buffer)。

对 `arei-mode' 调用包裹 condition-case:minor-mode 的故障不应中断
`scheme-mode-hook' 的后续步骤(尤其 font-lock 初始化)。"
  (unless (string-prefix-p " *org-src-fontification:" (buffer-name))
    (condition-case err
        (arei-mode 1)
      (error
       (message "[literal] arei-mode 启用失败,Scheme 染色不受影响: %S" err)))))
#+end_src

**** vertico-buffer 形态降级(bug#80812,同根因)

vertico-buffer 形态在 Emacs 31 master + Guix Emacs 30 编译的 vertico elc 下报 =vertico-buffer--setup: Wrong type argument: symbolp= 。根因与 arei 相同:Emacs 31 重构了 =setq-local= (基于新 =set-local= 函数,bug#80812),而 vertico-buffer 内部用的 =buffer-local-set-state= 宏在 30 编译的 elc 里展开不兼容。上游追踪:[[https://github.com/minad/vertico/issues/670][minad/vertico#670]]。修复前所有原本想用 =buffer= 的形态临时降级为 =indexed= ;vertico 上游修复或 Guix 用 Emacs 31 重编译后,在 [[#completion][Completion 框架]] 数据块的 =vertico-multiform-categories= 里把对应条目从 =indexed= 恢复为 =buffer= (如 =(consult-imenu indexed)= → =(consult-imenu buffer)= )。
