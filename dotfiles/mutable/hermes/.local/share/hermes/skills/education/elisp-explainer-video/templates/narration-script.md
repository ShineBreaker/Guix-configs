# 《<教程名>》视频解说稿模板

> 配音:`<voice_id>` / `<model>` / speed `<X>` / <原声|voice_modify>
> 受众:<概览|入门|手把手>(<详细度档位说明>)
> 语速参考:温柔学姐 speech-2.8-hd 1.0 速实测 **6.6 字/秒**
> 时长预算:880s 视频 → 入门档 5300-6200 字,留 80-100s 给画面/转场/章扉页

## 停顿标签(MiniMax t2a_v2 支持)

- `<#X#>` 在文本里塞这个,模型在那个位置停 X 秒(X ∈ [0.01, 99.99])
- 章扉页 / 章间过渡用 **1.5-2.0 秒**
- 句内换气 **0.3-0.5 秒**
- 段内主题切换 **0.6-1.0 秒**

## 语气词标签(仅 speech-2.8-hd / 2.8-turbo 支持)

`(laughs)` `(sighs)` `(breath)` `(coughs)` `(humming)` `(emm)` `(inhale)` `(exhale)` 等 18 个。
**不要滥用**——每章最多 1-2 个,纯讲解类可以全程不用。

## 三原则(稿子内化)

1. **画外音,不是念屏**。屏幕上画了什么,稿子里就**不**再说什么;只补画面没说的逻辑、动机、对比。
2. **英文术语保留**:M-x / use-package / vertico / init.el / company-mode / `(setq ...)` 之类不翻。硬翻会念得奇奇怪怪。
3. **每段 = 章扉页 5s(无音) + 内容 30-185s(有音)**。章扉页的"空"留给画面呼吸;音轨的起点 = 视频里章扉页的 `data-start`。

## 工作顺序(2026-06-23 沉淀)

1. **先出 7 段稿子(每章 1 个 ## Part N)**,总字数按目标档位估算
2. **合成 7 段 mp3**,记录实测时长(`ffprobe` 而非 t2a_v2 返回的 `audio_length`,边界更准)
3. **跟视频章边界对比**:每章音轨长 vs `[章起点, 下章起点]` 区间
4. **改稿子**:
   - 音轨 < 区间 → 加内容(白话定义 / 例子 / 新手问得到)
   - 音轨 > 区间 → 砍内容(重复定义 / 收尾客套)
   - **改稿子,不要改视频**(用户偏好 2026-06-23)
5. **再合成**,迭代到每章误差 < 5s
6. **章边界对齐烧录**:`scripts/align_and_merge.py`,溢出用 ffmpeg `-t` 裁切

## 章结构模板(每章复制一份)

```markdown
## Part N — <章主题>(视频 <start>-<end>s,目标 <秒数>s)

<开场过渡,接上一章结尾。1 句话,不重复上一章已经说过的内容。><#0.4#>
<本章核心概念 1 的白话定义。1-2 句,新人能听懂。><#0.4#>
<概念 1 的例子 / 怎么用 / 关键命令。3-5 句。><#0.4#>
<概念 2,如果本章有。><#0.4#>
<新手常见的坑或误解,1 个。><#0.3#>
<收束,留 1 句过渡到下一章。><#0.4#>
```

## 反模式(不要这样写)

- ❌ **念屏式**:"现在屏幕上显示的是 `(+ 2 2)`,它的意思是加法"——画面已经说了
- ❌ **超长复合句**:"Elisp 是一种基于 S-表达式的、源自 1958 年 McCarthy 设计的、Lisp 方言的、Emacs 专用的编程语言"——新人听完已经忘了开头
- ❌ **术语裸奔**:"现在我们用 `setq` 设置 `cursor-type`"——没说 `setq` 是什么
- ❌ **多义术语不区分**:"buffer 在这里是缓冲区,不是缓存"——一句话说不清就别用,换成"缓冲区"或"缓存"
- ❌ **结尾客套**:"以上就是今天的全部内容,希望对大家有所帮助"——不增加信息,删
