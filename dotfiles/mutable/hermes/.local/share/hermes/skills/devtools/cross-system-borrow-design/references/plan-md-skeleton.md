# PLAN.md 标准骨架(11 节固定)

> 这是 `cross-system-borrow-design` skill 推荐的 PLAN.md 骨架。
> 直接 copy 到目标文件,逐节填充即可。节标题不要改 —— 跨文档搜索
> 时靠节标题锚定(grep `^## 4\. 实施阶段` 一次能找所有这类文档)。

## 全局约定

- **行号引用格式**:`绝对路径:行号` 或 `绝对路径:行号-行号`
  - ✅ `/home/x/proj/src/foo.ts:42` 或 `src/foo.ts:42-78`
  - ❌ `foo.ts:42` (没绝对路径,grep 时找不到)
  - ❌ `foo.ts#L42` (URL 格式,grep 抓不住)
- **代码块语言标记**:JSON / YAML / SQL / TypeScript 必须标,让
  后续的 syntax highlighting / linter 友好
- **表格列宽**:不超过 4 列(超出时拆表)
- **图**:用 ASCII art,不引 mermaid(避免依赖)
