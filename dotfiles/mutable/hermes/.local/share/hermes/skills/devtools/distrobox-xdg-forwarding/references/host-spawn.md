# host-spawn notes & distrobox xdg-open wrapper structure

## host-spawn `-env` flag
`host-spawn [options] [COMMAND [args...]]`
Options:
- `-cwd <dir>`      change working directory of the spawned (host) process
- `-env <list>`     comma-separated env vars to pass to the host process.
                    **DEFAULT: `TERM`** — anything else (DISPLAY, WAYLAND_DISPLAY,
                    DBUS_SESSION_BUS_ADDRESS, XDG_RUNTIME_DIR) is DROPPED unless listed.
- `-no-pty` / `-pty`  allocate (or not) a pseudo-terminal for the host process
- `-version`

Implication: setting `DISPLAY=:0` in the container and then calling
`host-spawn xdg-open url` delivers `DISPLAY=[]` to the host, because DISPLAY is
not in the default `-env` whitelist. You must pass `-env TERM,DISPLAY,WAYLAND_DISPLAY,...`.

host-spawn does NOT do shell word-splitting on the command string. Passing
`host-spawn "env DISPLAY=:0 xdg-open"` makes it try to exec a binary literally
named `env DISPLAY=:0 xdg-open` → exit 127. Always pass the command as a single
token (`xdg-open`) plus separate args, and put env vars either via `-env` or via
`export` in the container script before the host-spawn call.

## distrobox `/usr/local/bin/xdg-open` wrapper (inside container)
Generated by distrobox at container init. Relevant structure:

- D-Bus remap block (near the host-spawn section):
  ```
  [ -z "${XDG_RUNTIME_DIR:-}" ] && XDG_RUNTIME_DIR="/run/user/$(id -ru)"
  [ -z "${DBUS_SESSION_BUS_ADDRESS:-}" ] && DBUS_SESSION_BUS_ADDRESS="unix:path=/run/user/$(id -ru)/bus"
  XDG_RUNTIME_DIR="/run/host/${XDG_RUNTIME_DIR}"
  DBUS_SESSION_BUS_ADDRESS="unix:path=/run/host/$(echo "${DBUS_SESSION_BUS_ADDRESS}" | cut -d '=' -f2-)"
  ```
  This remaps D-Bus to the host socket but does NOT touch DISPLAY/WAYLAND_DISPLAY.

- Two host-spawn invocation lines (the spots to patch):
  ```
  \tcd "${HOME}" > /dev/null 2>&1; host-spawn --no-pty "${host_command}" "$@"
  cd "${HOME}" > /dev/null 2>&1; host-spawn "${host_command}" "$@"
  ```
  `${host_command}` is the basename of the invoked command (e.g. `xdg-open`),
  `$@` are the args (the URL).

- The script uses `#!/bin/sh` and has `set -o nounset` — avoid referencing unset
  vars; write fixed values (`DISPLAY=:0`) rather than `${DISPLAY:-}`.

### Corrected lines (after patch)
```
\tcd "${HOME}" > /dev/null 2>&1; export DISPLAY=:0; export WAYLAND_DISPLAY=wayland-1; host-spawn --no-pty -env TERM,DISPLAY,WAYLAND_DISPLAY,DBUS_SESSION_BUS_ADDRESS,XDG_RUNTIME_DIR "${host_command}" "$@"
cd "${HOME}" > /dev/null 2>&1; export DISPLAY=:0; export WAYLAND_DISPLAY=wayland-1; host-spawn -env TERM,DISPLAY,WAYLAND_DISPLAY,DBUS_SESSION_BUS_ADDRESS,XDG_RUNTIME_DIR "${host_command}" "$@"
```

### Patching inside the container (prefer Python over sed)
The tabs + `${host_command}` + `"$@"` make `sed` via nested shell+sudo error-prone.
From the host, run a Python snippet inside the container:
```
distrobox-enter -nw <name> -- sudo python3 - <<'PY'
p="/usr/local/bin/xdg-open"
lines=open(p,encoding="utf-8").read().split("\n")
for i,l in enumerate(lines):
    if "host-spawn" not in l: continue
    l=l.replace('env DISPLAY=:0 WAYLAND_DISPLAY=wayland-1 ','')
    l=l.replace('export DISPLAY=:0; export WAYLAND_DISPLAY=wayland-1; ','')
    if "host-spawn --no-pty" in l:
        l=l.replace('host-spawn --no-pty "${host_command}"',
          'export DISPLAY=:0; export WAYLAND_DISPLAY=wayland-1; host-spawn --no-pty -env TERM,DISPLAY,WAYLAND_DISPLAY,DBUS_SESSION_BUS_ADDRESS,XDG_RUNTIME_DIR "${host_command}"')
    elif 'host-spawn "${host_command}"' in l:
        l=l.replace('host-spawn "${host_command}"',
          'export DISPLAY=:0; export WAYLAND_DISPLAY=wayland-1; host-spawn -env TERM,DISPLAY,WAYLAND_DISPLAY,DBUS_SESSION_BUS_ADDRESS,XDG_RUNTIME_DIR "${host_command}"')
    lines[i]=l
open(p,"w",encoding="utf-8").write("\n".join(lines))
PY
```
Verify with `distrobox-enter -nw <name> -- sudo sed -n '227,234p' /usr/local/bin/xdg-open | cat -A`
then end-to-end: `distrobox-enter -nw <name> -- bash -c "env -u DISPLAY -u WAYLAND_DISPLAY xdg-open https://example.com"` should open the host browser.

## Recurrence
Container rebuild / `distrobox create` / image pull rewrites `/usr/local/bin/xdg-open`,
wiping this patch. Re-run the justfile `patch-xdg-open` task (upgraded to the
`-env` form) after any recreate. The `register-scheme-handlers` task (for
`x-scheme-handler/*` like `zcode://`) is a separate concern and is unaffected.
